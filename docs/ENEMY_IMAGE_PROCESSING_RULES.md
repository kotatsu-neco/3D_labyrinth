# ENEMY_IMAGE_PROCESSING_RULES

この文書は、敵画像を本作の戦闘画面向けに処理するルールを定義する。
高解像度画像をそのまま使わず、意図的に低解像度・減色・粗い表示へ落とす。

## 1. 基本方針

- 敵画像は高解像度のまま戦闘画面に直接使わない。
- 本作では、古いDRPGの画面密度に合わせるため、敵画像をあえて十分に劣化させる。
- 画像の写実的な細部より、遠目で識別できる輪郭、姿勢、種別差を優先する。
- 元画像は制作素材として扱い、実行時に読み込む画像は低解像度版を基本とする。
- 敵画像の名称は仮名であり、敵名・固有名として正本化しない。

## 2. v21処理ルール

v21では、確認用の処理済み敵画像として以下を採用する。

- 出力形式: PNG
- 出力サイズ: 160x160 px
- 実質ディテール: 80px相当まで縮小後、最近傍で160pxへ拡大
- 色数目標: 24色
- 背景: 可能な範囲で透過
- 用途: 戦闘画面用候補・敵画像確認用
- 状態: ゲーム本体にはまだ組み込まない

## 3. 処理対象

- 通常敵画像: `assets/enemies/lowres/`
- シルエット画像: `assets/enemies/silhouettes_lowres/`
- 確認用一覧: `docs/assets/enemy_assets_lowres_contact_sheet_v21.png`
- 処理メタデータ: `data/enemy_assets_v21.json`

## 4. 劣化処理の意図

### 4.1 解像度を落とす理由

元画像は1254x1254 px程度で、現在のスマホ用DRPG画面には情報量が多すぎる。
そのまま使うと、画面上で現代的な高精細イラストに見え、黒地・白枠・低情報密度のUIと合わない。

### 4.2 減色する理由

色数を減らすことで、細部の写実感を弱め、ゲーム画面内の記号として扱いやすくする。

### 4.3 160x160にする理由

スマホ縦画面の戦闘ウィンドウ内で、複数敵やテキストUIと並べる余地を残すため。
必要に応じてCSS側で拡大表示しても、元画像の細部が戻らないようにする。

## 5. 背景透過

- 元画像に透明背景がある場合は維持する。
- 透明ではなくチェック柄背景が焼き込まれている画像は、白〜淡灰色の背景を可能な範囲で透過する。
- 自動処理のため、輪郭や明るい部位が一部欠ける可能性がある。
- 重要画像を正式採用する際は、必要に応じて個別調整する。

## 6. 採用前チェック

敵画像を正式採用する前に、以下を確認する。

- 160x160でも種別が判別できるか。
- 黒背景・暗いダンジョンUI上で見えるか。
- 輪郭が崩れすぎていないか。
- 低階層向け、中階層向け、深層向けの雰囲気と合っているか。
- 敵名が仮名のまま固定されていないか。
- 画像ファイル名と敵データIDが混同されていないか。

## 7. 禁止事項

- 高解像度敵画像を戦闘画面へ直接組み込まない。
- 画像ファイル名をそのまま敵の正式名称として扱わない。
- 敵名・階層名・固有名をAI案だけで正本化しない。
- 敵画像をゲーム本体へ組み込む前に、処理済み版と元画像を混在させない。
- 劣化処理済み画像を最終版と断定しない。
