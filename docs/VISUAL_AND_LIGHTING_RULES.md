# 表示・明暗設計の約束事

この文書は、灰の目録 3Dダンジョン試作における視認性、暗さ、松明・魔法光の扱いを固定するためのものです。

## 1. テスト表示と本番表示を分ける

開発中の視認性確認では、オブジェクト・壁・床・扉・階段の形状確認を優先し、必要に応じて明るめの表示を許容する。

一方、本番表示では、常に明るく見やすいダンジョンにはしない。ダンジョンは、視認性テスト版より気持ち暗めにしてよい。

## 2. 本番の暗さの方針

本番のダンジョンは、以下を目標とする。

- 暗いが、壁・床・扉・大きなオブジェクトは判別できる。
- 遠方や細部は見えにくくてよい。
- 見えにくさは、探索準備・緊張感・資源管理のために使う。
- 完全に何があるか分からない暗さにはしない。
- 暗さを、危険ヒントや露骨な誘導として使わない。

## 3. 松明・灯火・魔法光の前提

将来的に、松明・灯火・魔法によって視界を改善できる設計余地を残す。

想定:

- 通常状態: やや暗い。壁・床・扉・大きなオブジェクトは判別できる。
- 松明あり: 近距離と壁面が見やすくなる。
- 魔法光あり: 松明より安定して広く見える。床模様や小さな異変の発見にも関係させられる。
- 灯り切れ・暗闇状態: 遠方や細部が見えにくくなる。ただし操作不能や理不尽な不可視状態にはしない。

## 4. 実装上の準備

将来のため、以下の概念を実装に残せるようにする。

- `visibilityMode`: テスト表示と本番寄り表示を分けるための概念。
- `lightLevel`: 松明・魔法・暗闇状態の明るさを扱うための概念。
- 距離減衰: 遠方を暗くし、近距離は最低限判別できるようにする。
- 光源種別: 通常光、松明、魔法光、特殊暗闇などを後で分岐できるようにする。

## 5. 禁止事項

- テスト中の明るさを、そのまま本番仕様として固定しない。
- 本番で、常に均一で明るいダンジョンにしない。
- 暗すぎて壁・扉・階段・主要オブジェクトが判別できない状態を標準にしない。
- 暗さを使って、危険アイコンや露骨なヒント文の代わりにしない。
- 松明や魔法光の存在を考慮せず、視認性だけで明るさを決めない。

## 6. 検査観点

明暗を調整した場合は、以下を確認する。

- テスト表示では、形状確認に支障がないか。
- 本番寄り表示では、明るすぎず、暗すぎないか。
- 松明・魔法光なしでも、大きな構造物は判別できるか。
- 松明・魔法光ありの状態で、探索準備の効果を実感できる余地があるか。
- 暗さが雰囲気を作っているか。危険の事前ヒントになっていないか。
