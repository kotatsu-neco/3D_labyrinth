# UI雰囲気・イベントウィンドウ設計上の約束事

この文書は、灰の目録 3Dダンジョン試作におけるイベントウィンドウとUI雰囲気の設計ルールを固定するためのものです。

## 1. 目標

FC版Wizardry 1/2/3系の画面が持つ、以下の感覚をスマホ縦画面に移植する。

- 黒地を基調にする
- 枠線で情報領域を分ける
- 説明文を増やさず、選択肢で進める
- コマンドリストを縦に並べる
- パーティ情報を、状況判断の下地として近くに置く
- 余白と沈黙を残す

完全な模倣ではなく、固定グリッド式3Dダンジョンとスマホ操作に合う形へ再構成する。

## 2. 宝箱イベントの基本構造

宝箱イベントウィンドウは、原則として以下の構造にする。

1. 上段: メッセージ枠
2. 中段左: オブジェクト表示枠
3. 中段右: コマンド枠
4. 下段: パーティ情報枠

この構成により、イベント対象、現在の問い、選択肢、パーティ状態を一画面内で分離する。

## 3. 文言の扱い

イベントウィンドウでは、説明文を増やさない。

禁止:

- 宝箱の雰囲気説明を長く書くこと
- 罠や危険を事前に匂わせること
- プレイヤーが想像できる情報を文章で埋めること
- 状態を過剰に列挙すること

許容:

- `どうしますか`
- `誰が調べますか`
- `誰が解除しますか`
- `罠を発見した。`
- `解除した。`
- `解除に失敗した。`
- `開いた。`
- `開かなかった。`

## 4. コマンドの扱い

コマンドは縦リストを基本とする。

- 2列ボタンは避ける
- コマンドの左にカーソル記号を置く
- 文字は大きくしすぎない
- タップ領域はスマホ向けに確保する
- 無効ボタンを並べて状態説明に使わない

## 5. パーティ情報の扱い

パーティ情報は、宝箱や罠解除の判断に関係するため、イベントウィンドウ内に簡易表示してよい。

ただし、表示しすぎない。

- 名前
- 職業
- HP

程度に抑える。

## 6. スマホ用の落とし込み

FC版の画面比率をそのまま再現しない。

スマホ縦画面では、以下を優先する。

- タップできる行高
- 文字の読みやすさ
- 枠の役割が分かること
- 画面全体が重くなりすぎないこと

## 7. 検査観点

イベントウィンドウを変更した場合は、以下を確認する。

- メッセージと選択肢の位置関係が自然か
- 選択肢がコマンドリストとして読めるか
- 余計な説明文が増えていないか
- パーティ情報が邪魔になっていないか
- スマホで押しやすいか
- Wizardry的な黒地・枠線・沈黙が残っているか
