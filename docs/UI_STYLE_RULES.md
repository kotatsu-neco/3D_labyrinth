# UI雰囲気・イベントウィンドウ設計上の約束事

この文書は、タイトル未定 3Dダンジョン試作におけるイベントウィンドウとUI雰囲気の設計ルールを固定するためのものです。

## 1. 目標

FC版Wizardry 1/2/3系の画面が持つ、以下の感覚をスマホ縦画面に移植する。

- 黒地を基調にする
- 枠線で情報領域を分ける
- 説明文を増やさず、選択肢で進める
- コマンドリストを縦に並べる
- パーティ情報を、状況判断の下地として近くに置く
- 余白と沈黙を残す

完全な模倣ではなく、固定グリッド式3Dダンジョンとスマホ操作に合う形へ再構成する。

## 2. 通常探索画面の基本構造

通常探索画面は、以下を基本構造とする。

1. 3Dダンジョン表示
2. パーティ状態表示
3. 基本コマンド / 移動ボタン

通常探索画面に、常設の大きなメッセージ枠は置かない。
軽いフィードバックは必要最小限の一時表示に留める。

## 3. 宝箱イベントの基本構造

宝箱イベントウィンドウは、原則として以下の構造にする。

1. 上段: メッセージ枠
2. 中段左: オブジェクト表示枠
3. 中段右: コマンド枠
4. 下段: パーティ情報枠

この構成により、イベント対象、現在の問い、選択肢、パーティ状態を一画面内で分離する。

## 4. 文言の扱い

イベントウィンドウでは、説明文を増やさない。

禁止:

- 宝箱の雰囲気説明を長く書くこと
- 罠や危険を事前に匂わせること
- プレイヤーが想像できる情報を文章で埋めること
- 状態を過剰に列挙すること

許容:

- `どうしますか`
- `誰が調べますか`
- `誰が解除しますか`
- `罠を発見した。`
- `解除した。`
- `解除に失敗した。`
- `開いた。`
- `開かなかった。`


通常探索中の一時表示は、単語だけに削りすぎない。

方針:

- 余計な風景描写、危険の匂わせ、未確認の推測は書かない。
- ただし、プレイヤーが行った操作と、実際に起きた事実は書く。
- 「レバー。」のような対象名だけの表示は避ける。
- 「レバーを下げた。」「レバーを上げた。」「どこかで音がした。」のように、操作結果を短く明示する。

## 5. コマンドの扱い

コマンドは縦リストを基本とする。

- 2列ボタンは避ける
- コマンドの左にカーソル記号を置く
- 文字は大きくしすぎない
- タップ領域はスマホ向けに確保する
- 無効ボタンを並べて状態説明に使わない

## 6. パーティ情報の扱い

パーティ情報は、通常探索・宝箱・罠解除・戦闘判断に関係する基礎情報として扱う。

通常探索画面では、3列2行のカード型を基本候補とする。

通常カードの表示項目:

- 名前
- 職業
- HP
- AC
- 状態

MPや呪文回数は通常カードに表示しない。
魔法は7レベル別使用回数制とし、呪文回数はキャラクター詳細、魔法、キャンプなどの専用ウィンドウで扱う。

イベントウィンドウ内のパーティ表示も、通常探索画面と同じ情報体系にそろえる。

## 7. スマホ用の落とし込み

FC版の画面比率をそのまま再現しない。

スマホ縦画面では、以下を優先する。

- タップできる行高
- 文字の読みやすさ
- 枠の役割が分かること
- 画面全体が重くなりすぎないこと
- 3D表示領域を不必要に圧迫しないこと

## 8. 検査観点

UIを変更した場合は、以下を確認する。

- 通常探索画面に常設メッセージ枠が残っていないか
- 3D表示領域が狭くなりすぎていないか
- パーティカードの文字がスマホで読めるか
- ACが表示され、MPや呪文回数が通常カードに出ていないか
- メッセージと選択肢の位置関係が自然か
- 選択肢がコマンドリストとして読めるか
- 余計な説明文が増えていないか
- スマホで押しやすいか
- Wizardry的な黒地・枠線・沈黙が残っているか

## v16追加: キャラクター詳細の表示方針

キャラクター詳細では、通常探索カードに載せない情報を表示する。

表示対象:

- LEVEL
- ALIGN / RACE / CLASS
- HITS / AC / STATUS / GOLD
- STR / I.Q. / PIE / VIT / AGI / LUC
- MAGE / PRIEST の7レベル別呪文使用回数
- ITEMS

通常探索画面には、呪文使用回数やMP相当の情報を出さない。
詳細画面でも、MPという語は使わない。

## v17追記: キャンプ内画面の表示密度

キャンプ内のサブ画面は、通常探索画面へ情報を常駐させず、専用ウィンドウで必要な情報を確認する。

- 隊列画面では、順番・名前・職業・前衛/後衛を優先する。
- 呪文画面では、MAGE / PRIEST の7レベル別使用回数だけを表示する。
- アイテム画面では、所持品名を優先する。
- 装備確認画面では、実際の装備変更を行わず、所持品概要だけを表示する。

未実装機能については、説明を増やしすぎず、事実として「まだ行わない」と短く示す。

## v18追記: 探索イベントウィンドウ

探索イベントウィンドウは、宝箱イベントと同じUI文法で扱う。

- 黒地
- 白枠
- 上段メッセージ枠
- 左側の簡易ビジュアル枠
- 右側の縦コマンド

文言は、対象名だけにしない。
ただし、余計な風景描写や危険の匂わせも加えない。

良い例:

```text
祭壇がある。
石像を調べた。
床の紋様を調べた。
```

避ける例:

```text
祭壇。
石像。
何か不吉な気配がする。
```
