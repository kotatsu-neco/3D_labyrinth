# UI雰囲気・イベントウィンドウ設計上の約束事

この文書は、灰の目録 3Dダンジョン試作におけるイベントウィンドウとUI雰囲気の設計ルールを固定するためのものです。

## 1. 目標

FC版Wizardry 1/2/3系の画面が持つ、以下の感覚をスマホ縦画面に移植する。

- 黒地を基調にする
- 枠線で情報領域を分ける
- 説明文を増やさず、選択肢で進める
- コマンドリストを縦に並べる
- パーティ情報を、状況判断の下地として近くに置く
- 余白と沈黙を残す

完全な模倣ではなく、固定グリッド式3Dダンジョンとスマホ操作に合う形へ再構成する。

## 2. 通常探索画面の基本構造

通常探索画面は、以下を基本構造とする。

1. 3Dダンジョン表示
2. パーティ状態表示
3. 基本コマンド / 移動ボタン

通常探索画面に、常設の大きなメッセージ枠は置かない。
軽いフィードバックは必要最小限の一時表示に留める。

## 3. 宝箱イベントの基本構造

宝箱イベントウィンドウは、原則として以下の構造にする。

1. 上段: メッセージ枠
2. 中段左: オブジェクト表示枠
3. 中段右: コマンド枠
4. 下段: パーティ情報枠

この構成により、イベント対象、現在の問い、選択肢、パーティ状態を一画面内で分離する。

## 4. 文言の扱い

イベントウィンドウでは、説明文を増やさない。

禁止:

- 宝箱の雰囲気説明を長く書くこと
- 罠や危険を事前に匂わせること
- プレイヤーが想像できる情報を文章で埋めること
- 状態を過剰に列挙すること

許容:

- `どうしますか`
- `誰が調べますか`
- `誰が解除しますか`
- `罠を発見した。`
- `解除した。`
- `解除に失敗した。`
- `開いた。`
- `開かなかった。`

## 5. コマンドの扱い

コマンドは縦リストを基本とする。

- 2列ボタンは避ける
- コマンドの左にカーソル記号を置く
- 文字は大きくしすぎない
- タップ領域はスマホ向けに確保する
- 無効ボタンを並べて状態説明に使わない

## 6. パーティ情報の扱い

パーティ情報は、通常探索・宝箱・罠解除・戦闘判断に関係する基礎情報として扱う。

通常探索画面では、3列2行のカード型を基本候補とする。

通常カードの表示項目:

- 名前
- 職業
- HP
- AC
- 状態

MPや呪文回数は通常カードに表示しない。
魔法は7レベル別使用回数制とし、呪文回数はキャラクター詳細、魔法、キャンプなどの専用ウィンドウで扱う。

イベントウィンドウ内のパーティ表示も、通常探索画面と同じ情報体系にそろえる。

## 7. スマホ用の落とし込み

FC版の画面比率をそのまま再現しない。

スマホ縦画面では、以下を優先する。

- タップできる行高
- 文字の読みやすさ
- 枠の役割が分かること
- 画面全体が重くなりすぎないこと
- 3D表示領域を不必要に圧迫しないこと

## 8. 検査観点

UIを変更した場合は、以下を確認する。

- 通常探索画面に常設メッセージ枠が残っていないか
- 3D表示領域が狭くなりすぎていないか
- パーティカードの文字がスマホで読めるか
- ACが表示され、MPや呪文回数が通常カードに出ていないか
- メッセージと選択肢の位置関係が自然か
- 選択肢がコマンドリストとして読めるか
- 余計な説明文が増えていないか
- スマホで押しやすいか
- Wizardry的な黒地・枠線・沈黙が残っているか
