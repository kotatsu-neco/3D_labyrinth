# REPORT_v22a

## 概要

v22aでは、v22の全件検査で指摘した小修正を反映した。
主目的は、敵遭遇ウィンドウ試作のデモデータと外部JSON参照データの関係を明確にすること。
戦闘計算、経験値、戦利品、ランダム遭遇はまだ実装していない。

## 変更内容

- `app.js` 内の `ENCOUNTER_DEMOS` に、実機確認用の一時的なUIデモデータであることを明記。
- `ENCOUNTER_DEMO_DATA_SOURCE = "data/encounters_v22.json"` を追加。
- アプリ内デモ敵データに `enemyId` を追加。
  - `enemy_black_slime_proto`
  - `enemy_black_wing_bug_proto`
  - `enemy_goblin_raider_proto`
  - `enemy_bone_servant_proto`
- `data/encounters_v22.json` と `data/enemy_definitions_v22.json` に、v22a時点では参照用であり実行時読み込みではない旨を追記。
- v22全件検査レポートを `docs/reports/INSPECT_v22_v01.md` として同梱。
- キャッシュキーを `20260610-v22a` に更新。

## 変更していないこと

- 3D描画ロジック
- マップ構造
- 起動位置
- 宝箱、レバー、祭壇、石像、床の紋様、罠床の配置
- キャンプ、隊列、呪文、アイテム、キャラクター詳細の基本挙動
- 罠検知ロジック
- 宝箱イベントロジック
- 敵画像そのものの処理内容
- 敵遭遇ウィンドウの基本UI

## 確認済み

- `node --check app.js` 通過。
- モックDOM/WebGL環境で起動時実行チェック通過。
- 初回描画呼び出しチェック通過。
- 主要操作ボタン11件のイベント登録チェック通過。
- パーティカード6件の生成チェック通過。
- 位置表示が `B1F 浅層 x9 y10 W` になることを確認。
- `遭遇確認` ボタンから敵遭遇ウィンドウが生成されることを確認。
- 初回遭遇ウィンドウ内に敵画像枠が2件生成されることを確認。
- 全デモ遭遇3件で使うユニーク敵画像は4種類であることを確認。
- `data/enemy_definitions_v22.json` のJSON読み込み確認。
- `data/encounters_v22.json` のJSON読み込み確認。
- app.js内のデモ敵 `enemyId` が `data/enemy_definitions_v22.json` の敵定義IDに対応していることを確認。

## 未確認

- iPhone Safari実機表示。
- 敵遭遇ウィンドウの縦画面でのサイズ感。
- 低解像度敵画像の実機上での見え方。
- 戦闘コマンドの文言妥当性。
- 敵画像・敵分類名の最終採用可否。
- ランダム遭遇、戦闘計算、経験値、戦利品の接続。

## 注意

v22aの敵表示名は仮分類であり、敵名として正本化していない。
`戦う` / `身を守る` / `逃げる` は画面遷移・表示確認用であり、まだ戦闘結果には接続していない。
アプリ本体はまだ外部JSONを実行時読み込みしておらず、`ENCOUNTER_DEMOS` を使って表示確認している。
