# 灰の目録 3Dダンジョン試作 最新作業報告

## 現在版

- 最新試作: v16
- 主対象: キャラクター詳細ウィンドウ本格化 + キャンプ画面最小実装

## 目的

v15cで安定した通常探索画面を維持しつつ、Wizardry参考調査に基づいて、キャラクター詳細とキャンプ画面の接続を作る。

## 変更内容

- `index.html` のタイトルとキャッシュキーを v16 に更新。
- `キャンプ` ボタンを未実装トーストから専用キャンプウィンドウへ変更。
- キャンプウィンドウに以下のメニューを追加。
  - メンバーを見る
  - 隊列
  - 全員装備
  - 呪文
  - アイテム
  - 出る
- `メンバーを見る` からキャラクター選択画面へ進めるようにした。
- キャンプ経由で選んだキャラクターも、通常探索カードから選んだキャラクターも、同じ詳細ウィンドウで表示するようにした。
- キャラクターデータに以下を追加。
  - LEVEL
  - ALIGN
  - RACE
  - GOLD
  - STR / I.Q. / PIE / VIT / AGI / LUC
  - MAGE / PRIEST 7レベル別呪文使用回数
  - ITEMS
- キャラクター詳細ウィンドウに、能力値、呪文使用回数、所持品を表示するようにした。
- `docs/WIZARDRY_REFERENCE_NOTES.md` を同梱。
- `docs/SCREEN_FLOW_RULES.md` と `docs/UI_STYLE_RULES.md` をv16方針に合わせて更新。

## 維持した内容

- v15cで確認済みの3D表示・移動・旋回。
- 3列2行パーティカード。
- 通常探索カードは `名前 / 職業 / HP / AC / 状態` のまま。
- 通常探索画面には `MP` / `MAGIC` / 呪文回数を出さない。
- 宝箱イベントウィンドウ。
- 罠検知ON/OFF。
- 罠床は通常非表示、罠検知時だけ淡く表示。

## 検査結果

- `node --check app.js` 通過。
- 起動時実行チェック通過。
- 初回描画呼び出しチェック通過。
- 主要操作ボタンのイベント登録チェック通過。
- パーティカード6件の生成チェック通過。
- キャンプボタンからキャンプウィンドウを開けることをモックDOMで確認。
- キャンプ → メンバーを見る → キャラクター詳細 → 戻る、の経路をモックDOMで確認。
- 通常探索カード → キャラクター詳細、の経路をモックDOMで確認。
- `MP` / `MAGIC` がv16実UI文言として復活していないことを確認。
- zip破損チェック通過。
