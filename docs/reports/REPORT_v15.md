# 灰の目録 3Dダンジョン試作 v15 作業報告

## 現在版

- 最新試作: v15
- 主対象: 通常探索画面の基礎UI整理

## 目的

v14で実機動作確認済みとなった3Dダンジョン基盤を維持したまま、通常探索画面のUIを固める。

今回の主な判断は以下。

- 通常探索画面に常設の大きなメッセージ枠は置かない。
- イベント、戦闘、キャンプ、魔法等は専用ウィンドウで扱う。
- 通常探索画面のパーティ表示は、3列2行のカード型を試す。
- 魔法はMP制ではなく、7レベル別使用回数制を前提にする。
- 通常探索画面にはMPや呪文回数を出さない。
- ACは通常探索カードに表示する。

## 変更内容

- `index.html` のタイトルとキャッシュキーを v15 に更新。
- 通常探索画面から常設メッセージ枠を削除。
- 最小限の一時表示用 `statusToast` を追加。
- パーティ表示を3列2行カード型へ変更。
- 通常カードの表示項目を以下に整理。
  - 名前
  - 職業
  - HP
  - AC
  - 状態
- `MP` / `MAGIC` 表示を通常探索画面から削除。
- 宝箱イベントウィンドウ内のパーティ表示も `NAME / CLASS / HITS / AC / STATUS` に変更。
- キャラクターカードのタップで簡易ステータス専用ウィンドウを開くようにした。
- パーティ情報を `PARTY_MEMBERS` から描画する方式へ整理。
- 宝箱の罠解除成功担当を、盗賊の `ミラ` に修正。
- `docs/SCREEN_FLOW_RULES.md` を追加。
- `docs/COMBAT_RULES.md` を同梱。
- `docs/UI_STYLE_RULES.md` をv15方針に更新。

## 維持した内容

- v14で復旧した3D表示・移動・旋回ロジック。
- 罠検知ON/OFF処理。
- 罠床は通常非表示、罠検知時のみ淡く表示する仕様。
- 宝箱イベントウィンドウ。
- 宝箱は同じマスにいれば向きに関係なく調べられる仕様。
