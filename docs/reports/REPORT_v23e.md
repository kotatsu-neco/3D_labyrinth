# REPORT_v23e

## 概要

v23eでは、戦闘画面をキャラクター別コマンド入力前提で再構築した。v23dまでの全体コマンド方式を前提にしたレイアウトから、入力対象キャラクター、行動キュー、戻る、対象選択、呪文選択、道具選択を扱える骨格へ変更した。

また、戦闘LOGを通常縦積みレイアウトから外し、戦闘ウィンドウ内の浮動LOG小窓に変更した。

## 変更内容

- `docs/BATTLE_SCREEN_DESIGN_RULES.md` を同梱。
- `BUILD_VERSION` とキャッシュバスターを `v23e` に更新。
- 戦闘ウィンドウを以下の通常フローに再構成。
  - 敵表示エリア
  - パーティ状態エリア
  - 入力エリア
- LOGを通常フローから外し、`.battle-log-float` の浮動小窓に変更。
- キャラクター別入力状態を追加。
- 行動キューを追加。
- `攻撃` 選択後に敵対象選択へ進む骨格を追加。
- `守る` は即時行動キューへ登録。
- `呪文` / `道具` は選択画面へ遷移する骨格を追加。
- `戻る` で対象選択・呪文選択・道具選択から現在キャラのコマンドへ戻る処理を追加。
- `戻る` で前キャラクターの行動入力へ戻る処理を追加。
- 全員分の入力後に `決定 / 戻る` を表示する確認状態を追加。
- パーティ状態に、現在入力中/入力済み/未入力/行動不能を示す記号を追加。

## 変更していないこと

- 敵からの反撃は未実装。
- プレイヤーHPの戦闘中減少は未実装。
- 本式の逃走判定は未実装。
- 経験値・戦利品は未実装。
- 呪文・道具の本効果は未実装。
- 外部JSONの実行時読み込みは未実装。
