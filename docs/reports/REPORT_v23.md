# REPORT_v23

## 概要

v23では、敵遭遇ウィンドウに最小限の戦闘挙動を接続した。
具体的には、敵HPの生成、前衛3人による攻撃、命中時の敵HP減少、敵撃破表示までを実装した。
敵の反撃、プレイヤーHP減少、経験値、戦利品、ランダム遭遇はまだ実装していない。

## 変更内容

- `app.js` に `ENEMY_DEFINITIONS` を追加。
- 敵遭遇開始時に、敵グループごとの敵インスタンスを生成する処理を追加。
- 敵HPを `hp.dice` と `hp.fixed` から生成する処理を追加。
- `戦う` コマンドで、前衛3人が敵へ攻撃する処理を追加。
- 暫定命中判定を追加。
  - 本決定ではない。
  - ACは低いほど命中しにくいという原則だけを維持。
- 命中時に敵HPを減らす処理を追加。
- 敵HPが0になった個体を撃破扱いにする処理を追加。
- 全敵撃破時に、戦闘コマンドのうち `戦う` / `身を守る` / `逃げる` を無効化する処理を追加。
- 敵カードに、v23確認用として敵HPと残数を表示。
- `data/enemy_definitions_v23.json` を追加。
- `data/encounters_v23.json` を追加。
- `README.md` をv23内容へ更新。
- キャッシュキーを `20260610-v23` に更新。

## 変更していないこと

- 3D描画ロジック
- マップ構造
- 起動位置
- 宝箱、レバー、祭壇、石像、床の紋様、罠床の配置
- キャンプ、隊列、呪文、アイテム、キャラクター詳細の基本挙動
- 罠検知ロジック
- 宝箱イベントロジック
- 敵画像そのものの処理内容

- 初回遭遇ウィンドウ内に敵画像枠2件が生成されることを確認。

- 低解像度敵画像の実機上での見え方。

## 注意

v23の敵表示名は仮分類であり、敵名として正本化していない。
敵HP表示は確認用であり、本番戦闘画面で常時表示するとは限らない。
命中判定は暫定であり、`docs/COMBAT_RULES.md` の原則である「ACは低いほどよい」だけを反映している。
アプリ本体はまだ外部JSONを実行時読み込みしておらず、ローカル定数を使って表示確認している。
