# REPORT_v26

## 対象

- プロジェクト: 暗澹の石櫃 / Stone Casket of Gloom
- バージョン: v26
- ベース: v25

## 作業概要

v26では、v25までに接続した敵反撃・HP減少・暫定逃走判定・EXP/GOLD報酬を維持し、次段階としてランダム遭遇、戦利品、宝箱報酬、基礎的な呪文/道具効果、簡易レベルアップを接続した。

## 変更内容

### バージョン更新

- `app.js` の `BUILD_VERSION` を `v26` に変更。
- `index.html` の `<title>` を `暗澹の石櫃 v26` に変更。
- 画面上部HUDを `暗澹の石櫃 v26` に変更。
- キャッシュバスターを `20260612-v26` に変更。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v26` に更新。

### ランダム遭遇

- 移動アニメーション完了後にランダム遭遇判定を追加。
- イベント床、階段、オブジェクト上、戦闘/イベント表示中、全滅中はランダム遭遇しない。
- 最低歩数 `RANDOM_ENCOUNTER_MIN_STEPS = 4` 以降、`RANDOM_ENCOUNTER_STEP_CHANCE = 0.16` で発生する暫定仕様。
- 発生時は既存の `ENCOUNTER_DEMOS` からランダムに選ぶ。

### 戦利品

- ローカル定数 `TREASURE_TABLES` を追加。
- 各デモ遭遇に `treasureTableId` と `dropChance` を追加。
- 勝利報酬に `items` を追加。
- 戦利品は先頭の生存メンバーに付与する暫定仕様。
- アイテムはスタックせず、`items` 配列へ文字列を1件ずつ追加する。

### 宝箱報酬

- `chest01` に `treasureTableId: "shallow_chest_basic"` を追加。
- 解除済み宝箱を開いた時点で宝テーブルからアイテムを抽選し、先頭の生存メンバーに付与。
- 開封済み宝箱は既存どおり再取得しない。

### 呪文/道具効果

- `MAGE` Lv1を暫定攻撃呪文「火花」として処理。
- `PRIEST` Lv1を暫定回復呪文「治癒」として処理。
- `HEALING HERB` を戦闘中の回復道具として処理。
- 使用した `HEALING HERB` は所持品から削除する。

### 簡易レベルアップ

- `LEVEL_XP_THRESHOLDS` を追加。
- EXPが次レベル閾値以上になった場合、勝利報酬処理中にレベルアップ。
- 最大HPを増加し、HPを最大まで回復。
- MAG/PRI/BISはLv1呪文回数を暫定で増加。
