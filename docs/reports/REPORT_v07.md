# 灰の目録 3Dダンジョン試作 v07 作業報告

## 今回の修正対象

ユーザー指摘:

- 扉の色が明るすぎる。
- 扉の開く方向が右と左で互い違いに見える。
- 下り階段が下り階段としておかしい。
- 階段は角に置くと、将来の上り階段との作図上の整合性が取りにくい可能性がある。

## 変更内容

### 1. 扉色の調整

`app.js` の `doorColor` をv06より暗い茶色へ変更しました。

- v06: 明るめの単色茶色
- v07: 暗め・低彩度の茶色

まだら表現は復活させず、単色寄りのまま明度だけを下げています。

### 2. 開扉方向の統一

開いた後の左右扉板が互い違いに見えないよう、左右の扉板を同じ側へ倒す配置に変更しました。

対象:

- 南北方向の扉
- 東西方向の扉

### 3. 下り階段の移動

下り階段を角付近の `x10 y1` から、角ではない `x8 y8` へ移動しました。

理由:

- 角配置では開口部・側壁・下階接続方向が作図しにくい。
- 将来の上り階段との対応関係を作りにくい。
- 階段が床上の台に見えやすい。

### 4. 下り階段の床穴化

従来の階段は、標準床の上に段差オブジェクトを置く形でした。v07では以下に変更しました。

- 階段タイルでは標準床を描画しない。
- 床の周囲に縁を残す。
- 開口部の中に暗がりを置く。
- 段を床面より下に沈める。
- 左右に側壁を描画する。

これにより、床上の台ではなく、下層へ続く穴として見せる方向へ修正しました。

### 5. 設計ルール文書の同梱

`docs/DUNGEON_DESIGN_RULES.md` を同梱しました。

この文書には、以下の方針を記載しています。

- 階段は原則として角に置かない。
- 下り階段は床上オブジェクトではなく床開口部として扱う。
- 下り階段は将来の上り階段と対応できる前提で配置する。
