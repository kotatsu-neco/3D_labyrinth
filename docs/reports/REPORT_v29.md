# REPORT_v29

## 対象

- プロジェクト: 暗澹の石櫃 / Stone Casket of Gloom
- バージョン: v29
- ベース: v28

## 作業概要

v29では、v28で接続したキャンプ内アイテム操作を土台に、装備変更を追加した。

所持品は実行時にアイテム個体へ正規化し、同名アイテムでも別個体として扱う方針を維持した。キャンプ内のアイテム選択画面に `装備 / 外す` を追加し、装備中表示、装備可能/不可表示、有効AC、武器攻撃ダイスを接続した。

## 変更内容

### バージョン更新

- `app.js` の `BUILD_VERSION` を `v29` に変更。
- `index.html` の `<title>` を `暗澹の石櫃 v29` に変更。
- 画面上部HUDを `暗澹の石櫃 v29` に変更。
- キャッシュバスターを `20260612-v29` に変更。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v29` に更新。

### 装備定義

- `ITEM_DEFINITIONS` を追加し、v29時点の暫定装備対象を定義。
- 装備スロットは `weapon / armor / shield / head`。
- 武器は攻撃ダイスと最小限の命中補正を持つ。
- 防具・盾・兜はAC補正を持つ。
- ACは低いほど良い方針を維持し、装備補正は `-1`、`-2` など有効ACを下げる方向で扱う。

### 所持品個体化

- 既存の文字列所持品を、実行時に `{ instanceId, name, equippedSlot }` 形式へ正規化。
- 既存初期装備は、最初に見つかったスロットごとの装備可能アイテムを装備中として扱う。
- v28時点の初期AC表示が大きく変わらないよう、初期装備補正を差し引いて `baseAc` を設定。
- 戦利品・宝箱報酬で追加されるアイテムは、新しいアイテム個体として追加。

### キャンプ内装備操作

- キャンプメニューの `装備確認` を `装備` に変更。
- `装備` 画面でメンバーごとの装備概要と有効ACを表示。
- メンバー選択後、所持品画面へ遷移し、アイテムごとに `装備 / 外す` を実行できるようにした。
- アイテム一覧では、装備中は `E`、装備可能品は `WEAPON / ARMOR / SHIELD / HEAD`、使用可能品は `USE`、未接続品は `-` と表示。
- アイテム選択後の詳細表示に `TYPE / USE / EQUIP` を追加。

### 戦闘・ステータス反映

- `effectiveMemberAc(member)` を追加。
- 探索カード、キャンプ内メンバー選択、装備一覧、戦闘パーティ表示、ステータス詳細のAC表示を有効AC参照へ変更。
- 敵攻撃時の命中判定も有効ACを参照するように変更。
- `memberDamageDice(member)` で装備武器の攻撃ダイスを参照するように変更。
- `memberHitBonus(member)` で装備武器の暫定命中補正を参照するように変更。

### 受け渡し・破棄との整合

- 装備中アイテムを渡す場合、渡す前に装備状態を解除する。
- 装備中アイテムを捨てる場合、削除前に装備状態を解除する。
- 受け取り側に渡ったアイテムは未装備状態になる。
