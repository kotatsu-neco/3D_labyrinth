# REPORT_v23g

## 概要

v23gでは、v23fに対するユーザーのiPhone Safari実機確認結果を受けて、ステータス詳細画面と戦闘画面を追加調整した。

主眼は以下の3点。

1. ステータス詳細画面の所持アイテムを上寄せにする。
2. ステータス詳細画面にAGEを追加する。
3. 戦闘画面のコマンド部分とLOG小窓を広げ、スクリーンショット上の余裕を操作性に回す。

## 変更内容

### ステータス詳細画面

- 所持アイテム欄を、均等割付けではなく上寄せ表示に変更。
- 所持アイテムリストを `ul` に変更し、独自の中黒表示で項目を上から詰めて表示するようにした。
- `ALIGN / RACE / CLASS` の下に `AGE` を追加。
- パーティメンバーのプロトタイプ表示データに `age` を追加。
- 左カラムに基本情報・能力値、右カラムに所持アイテムというv23fの構造は維持。
- `ATTRIBUTES` ラベルは復活させていない。

### 戦闘画面

- 敵表示欄で敵カードが縦に伸びすぎないよう、敵カード領域を2列配置寄りに再調整。
- 下部コマンドパッドの高さとボタン高を拡大。
- LOG小窓の幅・高さを拡大。
- LOG本文領域を拡大。
- LOG小窓の `×` と再表示用 `LOG` ボタンは維持。
- LOGは通常レイアウト内の段には戻していない。
- コマンドは右カラムへ押し込まず、下部全幅の2列操作パッドとして維持。

## 確認済み

- `node --check app.js` 通過。
- `data/*.json` 全件パース成功。
- `BUILD_VERSION` が `v23g` であることを確認。
- `index.html` のタイトル/HUD表示が `v23g` であることを確認。
- `index.html` のキャッシュバスターが `20260612-v23g` であることを確認。
- `app.js` 内に `age` 表示用の `AGE` 行があることを確認。
- `style.css` 内でステータス詳細アイテム欄が `justify-content: flex-start` 指定であることを確認。
- `style.css` 内で `.battle-log-float` が `position: absolute` 指定であることを確認。
- `app.js` の通常コマンド表示に `quantity` / `weight` を導入していないことを確認。
- zip破損チェック通過。
