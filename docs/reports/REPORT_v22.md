# REPORT_v22

## 概要

v22では、v21で作成した低解像度・減色済み敵画像を使い、敵遭遇ウィンドウの最小試作を追加した。
戦闘計算、経験値、戦利品、ランダム遭遇はまだ実装していない。

## 変更内容

- 通常探索画面に `遭遇確認` ボタンを追加。
- 敵遭遇ウィンドウを追加。
  - 低解像度敵画像を表示。
  - 敵の仮分類名と出現数を表示。
  - `戦う` / `身を守る` / `逃げる` / `離れる` を表示。
- `data/enemy_definitions_v22.json` を追加。
  - 戦闘本実装前の敵定義プロトタイプ。
- `data/encounters_v22.json` を追加。
  - B1F浅層の仮遭遇テーブルと、アプリ内確認用遭遇データ。
- `docs/PARAMETER_DESIGN_RULES.md` を改訂版に更新。
  - アイテムはスタックさせない。
  - アイテムに物理重量 `weight` は持たせない。
  - 同じアイテムを複数持つ場合も、所持品インスタンスを複数作る。
- キャッシュキーを `20260610-v22` に更新。

## 変更していないこと

- 3D描画ロジック
- マップ構造
- 起動位置
- 宝箱、レバー、祭壇、石像、床の紋様、罠床の配置
- キャンプ、隊列、呪文、アイテム、キャラクター詳細の基本挙動
- 罠検知ロジック
- 宝箱イベントロジック
- 敵画像そのものの処理内容

## 確認済み

- `node --check app.js` 通過。
- モックDOM/WebGL環境で起動時実行チェック通過。
- 初回描画呼び出しチェック通過。
- 主要操作ボタン11件のイベント登録チェック通過。
- パーティカード6件の生成チェック通過。
- 位置表示が `B1F 浅層 x9 y10 W` になることを確認。
- `遭遇確認` ボタンから敵遭遇ウィンドウが生成されることを確認。
- 敵遭遇ウィンドウ内に敵画像枠が2件生成されることを確認。
- `data/enemy_definitions_v22.json` のJSON読み込み確認。
- `data/encounters_v22.json` のJSON読み込み確認。

## 未確認

- iPhone Safari実機表示。
- 敵遭遇ウィンドウの縦画面でのサイズ感。
- 低解像度敵画像の実機上での見え方。
- 戦闘コマンドの文言妥当性。
- 敵画像・敵分類名の最終採用可否。
- ランダム遭遇、戦闘計算、経験値、戦利品の接続。

## 注意

v22の敵表示名は仮分類であり、敵名として正本化していない。
`戦う` / `身を守る` / `逃げる` は画面遷移・表示確認用であり、まだ戦闘結果には接続していない。
