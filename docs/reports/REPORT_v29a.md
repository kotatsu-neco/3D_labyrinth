# REPORT_v29a

## 対象

- プロジェクト: 暗澹の石櫃 / Stone Casket of Gloom
- バージョン: v29a
- ベース: v29

## 作業概要

v29aでは、v29で実装した装備変更を全件検査し、実装ロジックを追跡した。検査中に、装備ロジック自体ではなく、装備入口から所持品画面へ入った場合の戻り先保持にUIフロー上の不整合を1件検出したため、修正した。

## 修正内容

- `BUILD_VERSION` を `v29a` に更新。
- `index.html` の `<title>` と画面上部HUDを `暗澹の石櫃 v29a` に更新。
- キャッシュバスターを `20260612-v29a` に更新。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v29a` に更新。
- `renderItemWindow(memberId, notice, returnMode)` を導入し、所持品画面の戻り先を保持できるようにした。
- `renderEquipAllWindow()` から所持品画面へ入る場合は `returnMode = "equipAll"` を渡すようにした。
- 所持品画面・アイテム詳細・使用対象・受け渡し・破棄確認の各画面で `returnMode` を引き回すようにした。
- 装備入口から入った所持品画面では、戻りボタンを `装備一覧へ戻る` とし、`renderEquipAllWindow()` へ戻すようにした。

## ロジック検査で確認したこと

- 初期所持品は実行時に `{ instanceId, name, equippedSlot }` 形式へ正規化される。
- 同一メンバー内で同一スロットの装備は1件に制限される。
- 初期有効ACはv28表示値を維持する。
- 武器を外すと、攻撃ダイスは職業別フォールバックに戻る。
- 武器を再装備すると、武器攻撃ダイスと命中補正が復帰する。
- 防具を外すと有効ACが変化し、再装備で復帰する。
- 同一スロットの別防具を装備すると、既存防具は自動で外れる。
- 装備中アイテムを渡す場合、渡す前に未装備化され、受け取り側でも未装備状態になる。
- 装備中アイテムを捨てる場合、削除前に未装備化される。
- `HEALING HERB` はアイテム個体化後も戦闘外使用でき、1個だけ消費される。
- 装備入口から所持品画面へ入った場合、戻りボタンは装備一覧へ戻る。
