# REPORT v31

## 概要

v31では、ユーザー決定事項に従い、ダンジョン内パーティ行動の基礎を拡張した。
主対象は、所持品上限、状態異常、宝箱の罠種別選択、罠発動、戦利品受け取りである。

## 実装内容

### 所持品上限

- 1キャラクターの所持品上限を10個にした。
- `addItemToMember()` は満杯時に `false` を返す。
- アイテム受け渡しでは、受け取り側が満杯の場合に渡せない。
- 宝箱戦利品は、選択した受け取り先に空きがなければ受け取れない。
- 戦闘報酬アイテムも所持上限を参照する。

### 状態異常

以下を状態定義として実装した。

- `OK`
- `POISONED`
- `ASLEEP`
- `AFRAID`
- `PARALYZED`
- `STONED`
- `OUT`
- `DEAD`
- `ASHES`
- `LOST`

状態定義により、以下の判定を統合した。

- 生存扱い
- 行動可能
- 攻撃対象になりうるか
- HP回復対象になりうるか
- 報酬対象になりうるか
- 歩行時ダメージ

`POISONED` は1歩移動するごとにHPが1減る。
旧`DOWN`表記は、内部正規化で`DEAD`へ変換する。

### 宝箱・罠

- 宝箱ごとに実行時状態を持つようにした。
- 罠未調査、罠推定、解除済み、開封済みを表示する。
- `調べる` では調査者を選ぶ。
- 調査成功時は実際の罠を推定表示する。
- 調査失敗時は誤った罠を推定表示する。
- `罠を解除する` では解除者を選んだ後、罠種別を選ぶ。
- 誤った罠種別を選ぶと罠が作動する。
- 解除失敗でも罠が作動する。
- 未解除のまま開けても罠が作動する。
- 開封後の戦利品は、受け取り先を選ぶ。

### 罠種別

- 毒針
- 眠りのガス
- 恐怖の刻印
- 麻痺針
- 石化の霧
- 石弓
- 爆発
- 灰化の呪い
- 喪失の封印

## VMテストで確認したこと

- 所持品上限が10個である。
- 10個を超えるアイテム追加は拒否される。
- 満杯のメンバーへアイテムを渡せない。
- `POISONED` は1歩分の処理でHPが1減る。
- `PARALYZED` は行動不可になる。
- 宝箱画面に未調査状態が表示される。
- 調査後、宝箱状態がcheckedになる。
- 解除者選択後、罠種別選択画面が出る。
- 罠種別選択画面に全罠候補が出る。
