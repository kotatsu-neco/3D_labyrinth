# REPORT v32

## 概要

v32では、v31で実装した宝箱・罠・所持品上限・状態異常基盤を、戦闘勝利後の処理へ接続した。
主対象は、戦闘勝利後の戦利品自動取得をやめ、罠付き宝箱として調査・解除・開封・受け取り先選択へ流すことである。

## 実装内容

### 戦闘後宝箱

- 戦闘ごとに `instanceId` を持たせた。
- 戦闘勝利時、EXP/GOLDは従来通り配分する。
- 戦闘勝利時のアイテム報酬は自動取得しない。
- アイテム報酬がある場合、戦闘後宝箱の固定戦利品として保持する。
- 戦闘結果画面に `CHEST` 状態を表示する。
- 未開封の戦闘後宝箱がある場合、結果画面に `宝箱へ` と `立ち去る` を表示する。
- `宝箱へ` を押すと、宝箱画面へ遷移する。
- 戦闘後宝箱にも罠IDを割り当てる。
- 戦闘後宝箱は、v31で実装した調査・解除・罠種別選択・罠発動・受け取り先選択・所持上限判定をそのまま使う。
- 戦闘後宝箱を閉じると探索へ戻る。

### 既存仕様の維持

- 1人10個の所持品上限は維持。
- 全状態異常の基盤は維持。
- 毒歩行ダメージは維持。
- 固定配置宝箱の処理は維持。
- キャンプ、装備、アイテム、戦闘外治癒は維持。

## VMテストで確認したこと

- 戦闘報酬アイテムは勝利時に自動取得されない。
- アイテム報酬がある場合、戦闘結果画面に `宝箱へ` が表示される。
- `createPostBattleChest()` で戦闘後宝箱が生成される。
- 戦闘後宝箱の固定戦利品は、戦闘報酬アイテムと一致する。
- 戦闘後宝箱はv31の宝箱UIで表示される。
- 戦闘後宝箱の戦利品は受け取り先選択を経由して取得される。
- 戦闘後宝箱の閉じる処理で探索へ戻る。
