# REPORT_v28

## 対象

- プロジェクト: 暗澹の石櫃 / Stone Casket of Gloom
- バージョン: v28
- ベース: v27

## 作業概要

v28では、v27までの戦闘中対象選択を維持したうえで、キャンプ内アイテム操作を追加した。

これまで「使用・受け渡し・破棄はまだ行わない」と表示していたアイテム画面を、アイテム選択式の操作画面に変更した。戦闘外の `HEALING HERB` 使用、メンバー間の受け渡し、確認付き破棄を接続した。

## 変更内容

### バージョン更新

- `app.js` の `BUILD_VERSION` を `v28` に変更。
- `index.html` の `<title>` を `暗澹の石櫃 v28` に変更。
- 画面上部HUDを `暗澹の石櫃 v28` に変更。
- キャッシュバスターを `20260612-v28` に変更。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v28` に更新。

### キャンプ内アイテム操作

- キャンプのアイテム画面で、所持品1件ごとに選択できるUIへ変更。
- アイテム選択後に `使う / 渡す / 捨てる / 戻る` を表示。
- 未接続アイテムは `USE` 欄を `-` として表示。
- `HEALING HERB` は `USE` と表示し、戦闘外使用の対象にした。

### 戦闘外HEALING HERB使用

- `HEALING HERB` を生存中かつ負傷しているメンバーに使用可能にした。
- 回復量は暫定で `1d6+3`。
- 使用後、該当アイテム1件だけを所持品から削除する。
- 回復対象がいない場合は対象ボタンを無効化し、消費されない。
- `DOWN` 状態のメンバーは対象外。蘇生はまだ接続していない。

### アイテム受け渡し

- 所持アイテムを別メンバーへ渡せるようにした。
- 同一メンバーへの受け渡しは無効化。
- アイテムはスタックせず、受け取り側の所持品末尾に1件追加する。

### アイテム破棄

- `捨てる` 操作を追加。
- 誤操作を避けるため、破棄前に確認画面を挟む。
- 確認後、該当アイテム1件だけを所持品から削除する。
