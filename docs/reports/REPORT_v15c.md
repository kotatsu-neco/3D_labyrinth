# 灰の目録 3Dダンジョン試作 v15c 作業報告

## 現在版

- 最新試作: v15c
- 主対象: v15全件検査後の修正整理 + v15b実機確認後の文言修正

## 目的

v15で導入した通常探索画面のカード型UIを維持しつつ、全件検査で見つかった問題とv15b実機確認後の文言問題を修正する。

- 3D表示領域を広げたという報告とCSS実態の差をなくす。
- 起動直後の説明トーストを削除する。
- 通常探索中の一時表示文言を、短すぎる単語表示ではなく、操作結果と確認事実が分かる短文にする。
- 簡易マップCSSを動的追加から静的CSSへ移す。

## 変更内容

- `index.html` のタイトルとキャッシュキーを v15c に更新。
- `style.css` の3D表示領域を以下へ変更。
  - `grid-template-rows: minmax(clamp(300px, 46svh, 360px), 1fr) auto auto;`
  - `.viewport-panel` / `#dungeonCanvas` の最小高さも同じ基準に統一。
- 起動直後に出していた `v15: 通常探索画面をカード型パーティ表示へ変更しました。` のトーストを削除。
- 通常探索中の一時表示文言を、事実が分かる短文へ修正。
  - `進めない。`
  - `レバーを下げた。……どこかで音がした。`
  - `レバーを上げた。……どこかで音がした。`
  - `扉を開けた。`
  - `石壁だ。`
  - `下り階段がある。`
  - `床に紋様がある。`
  - `何も見つからない。`
- 移動・旋回時の逐次トーストを削除。
- `.map-overlay` のCSSを `style.css` に移し、`app.js` から動的 `<style>` 挿入を削除。

## 維持した内容

- v14で復旧した3D表示・移動・旋回ロジック。
- 罠検知ON/OFF処理。
- 罠床は通常非表示、罠検知時のみ淡く表示する仕様。
- 宝箱イベントウィンドウ。
- 宝箱は同じマスにいれば向きに関係なく調べられる仕様。
- v15で導入した3列2行パーティカード。
- 通常カードの `名前 / 職業 / HP / AC / 状態` 表示。
- `MP` / `MAGIC` を通常探索画面と宝箱イベント内パーティ表示に出さない方針。
