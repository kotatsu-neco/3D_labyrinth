# BATTLE_SCREEN_DESIGN_RULES

この文書は、戦闘画面を場当たりに作らないための設計ルールである。キャラクター別コマンド入力、キャンセル、対象選択、呪文選択、道具選択、浮動ログ小窓を前提に、スマホ縦画面で破綻しにくい完成形から逆算する。

## 1. 固定する前提

- 戦闘は専用ウィンドウで扱う。
- 戦闘中は通常探索操作を受け付けない。
- 戦闘ログは通常縦積みレイアウトに参加させず、独立した浮動小窓として扱う。
- 戦闘コマンドはキャラクター別入力を前提にする。
- キャラクター別入力では、戻る、キャンセル、対象選択、呪文選択、道具選択が必ず発生する。
- 全体コマンドだけを基準にレイアウトしない。
- 戦闘画面に大見出し枠を置かない。
- コマンドを右カラムへ押し込まない。

## 2. 戦闘の基本フロー

```text
敵が現れる
  ↓
1人目の行動入力
  ↓
必要なら対象選択 / 呪文選択 / 道具選択
  ↓
2人目以降の行動入力
  ↓
全員分の行動入力完了
  ↓
行動確認
  ↓
ラウンド実行
  ↓
ログ表示
  ↓
敵全滅 / パーティ全滅 / 逃走 / 次ラウンド
```

## 3. 画面構造

通常フローは以下の3層に限定する。

```text
1. 敵表示エリア
2. パーティ状態エリア
3. 入力エリア
```

LOGはこの通常フローに入れない。戦闘ウィンドウ内の浮動レイヤーとして表示する。

禁止する構造:

```text
敵表示
LOG
パーティ状態
コマンド
```

採用する構造:

```text
戦闘ウィンドウ
  ├─ 敵表示エリア
  ├─ パーティ状態エリア
  ├─ 入力エリア
  └─ 浮動LOG小窓
```

## 4. 入力状態

入力エリアは状態ごとに差し替える。

- `character_command`: キャラクター別コマンド選択。
- `target_select`: 攻撃・呪文・道具の対象選択。
- `spell_select`: 呪文選択。
- `item_select`: 道具選択。
- `round_confirm`: 全員入力後の確認。
- `round_resolving`: ラウンド実行中。
- `battle_result`: 戦闘結果表示。

## 5. キャラクター別コマンド

初期候補:

- 攻撃
- 守る
- 呪文
- 道具
- 逃げる
- 戻る

呪文は使用可能職のみ表示する。道具は所持品がある場合に表示する。後衛攻撃や射程の扱いは後続で決定する。

## 6. 戻る・キャンセル

戻る操作なしにキャラクター別コマンドを実装しない。

戻るには以下を含む。

- 対象選択から現在キャラクターのコマンド選択へ戻る。
- 呪文選択から現在キャラクターのコマンド選択へ戻る。
- 道具選択から現在キャラクターのコマンド選択へ戻る。
- 現在キャラクターの行動選択を取り消し、前のキャラクターへ戻る。
- 全員入力後の確認から、最後のキャラクター入力へ戻る。

## 7. 行動キュー

各キャラクターの選択済み行動は、行動キューに保存する。

```json
[
  { "characterId": "adel", "command": "attack", "target": "enemy_inst_0001" },
  { "characterId": "mira", "command": "defend", "target": null }
]
```

戻る場合は、行動キューの最後の要素を削除し、そのキャラクターを再入力対象にする。

## 8. パーティ状態表示

戦闘中は以下を優先する。

- 名前
- HP
- AC
- 状態
- 現在入力中 / 入力済み / 未入力 / 行動不能

呪文回数や所持品一覧は戦闘パーティ状態欄に出さない。

## 9. 浮動LOG小窓

- LOGは通常フローの1段ではない。
- 入力エリア直上に見える位置へ浮かせる。
- 入力エリアを押し下げない。
- 半透明黒背景、白枠、高さ固定。
- 文字送り速度は内部定数で調整可能。
- LOG小窓全体をタップ対象にする。
- 文字送り中にタップした場合は即時全文表示する。
- LOG表示中にコマンドを押した場合は、まずLOG全文表示を優先し、その押下では行動を進めない。

## 10. v23eで行うこと

- 戦闘画面の構造を再構築する。
- LOGを通常フローから外し、浮動LOG小窓にする。
- 入力エリアをキャラクター別コマンド前提の構造にする。
- 1人ずつ入力が進む骨格、戻る、行動キュー、ラウンド実行の最小形を作る。
- 呪文・道具・敵反撃・逃走判定・報酬処理は本格実装しない。

## 11. 禁止事項

- キャラクター別入力を前提にせず、全体コマンドだけを基準にレイアウトしない。
- キャンセルや戻るを後付けにしない。
- LOGを通常レイアウト内の段として置かない。
- コマンドを右カラムへ押し込まない。
- 大見出しで戦闘画面の縦幅を消費しない。
- ボタンやリストが画面外へ押し出される構造にしない。
- モックDOMでの確認をiPhone Safari実機確認済みと書かない。
