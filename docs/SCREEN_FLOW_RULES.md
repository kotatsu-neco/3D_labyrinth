# 画面遷移・専用ウィンドウ設計上の約束事

この文書は、本作における通常探索画面、イベント、戦闘、キャンプ等の表示単位を固定するためのものです。

## 1. 基本方針

通常探索画面は、常時表示される基礎画面とする。

通常探索画面には、以下を置く。

- 3Dダンジョン表示
- パーティ状態表示
- 基本コマンド
- 移動ボタン

通常探索画面に、常設の大きなメッセージ枠は置かない。

## 2. 専用ウィンドウ方式

イベント、戦闘、キャンプ、隊列、アイテム、魔法などは、通常探索画面へ直接詰め込まず、原則として専用ウィンドウを前面に開く。

想定:

```text
通常探索画面
  ↓
宝箱イベントウィンドウ
  ↓
石像・祭壇イベントウィンドウ
  ↓
会話イベントウィンドウ
  ↓
戦闘ウィンドウ
  ↓
キャンプ / 隊列 / アイテム / 魔法ウィンドウ
```

## 3. 専用ウィンドウ表示中の入力

専用ウィンドウ表示中は、通常探索操作を止める。

- 前進しない。
- 後退しない。
- 左右旋回しない。
- 調べるを通常探索処理として受け付けない。
- ウィンドウ内のコマンドだけを受け付ける。

## 4. メッセージの扱い

通常探索画面には常設メッセージ枠を置かない。

通常探索中の軽いフィードバックは、必要最小限の一時表示に留める。
ただし、短ければよいわけではない。対象名だけではなく、操作結果または確認した事実が分かる短文にする。

例:

- 初期位置に戻った。
- 罠検知を有効にした。
- 進めない。
- レバーを下げた。……どこかで音がした。
- 扉を開けた。
- 石壁だ。

イベントの本文、選択肢、結果は専用ウィンドウ内に表示する。

## 5. パーティ表示

通常探索画面のパーティ表示は、3列2行のカード型を基本候補とする。

カード表示項目:

- 名前
- 職業
- HP
- AC
- 状態

MPや呪文回数は通常探索カードに表示しない。
呪文回数は、キャラクター詳細、魔法、キャンプなどの専用ウィンドウで扱う。

## 6. キャラクターカード

キャラクターカードは、将来的にキャラクター詳細ウィンドウへのショートカットにする。

v15では、カードタップで簡易ステータスウィンドウを開く。

## 7. 禁止事項

- 通常探索中の一時表示を「レバー。」のような対象名だけにしない。
- 通常探索画面へイベント本文や戦闘情報を過剰に常駐させない。
- メッセージ枠を画面中央に常設して3D表示領域を圧迫しない。
- イベント・戦闘・キャンプ・魔法の情報を通常探索画面へまとめて詰め込まない。
- 専用ウィンドウ表示中に通常移動操作が通ってしまう状態にしない。

## v16追加: キャンプとキャラクター詳細

v16では、キャンプ画面とキャラクター詳細画面の最小接続を実装する。

```text
通常探索画面
  ↓ キャンプ
キャンプウィンドウ
  ↓ メンバーを見る
メンバー選択
  ↓ キャラクター選択
キャラクター詳細
```

通常探索画面のキャラクターカードからも、同じキャラクター詳細を開ける。

v16では、装備変更・魔法使用・アイテム使用・隊列変更の本処理は実装しない。
未実装項目は、専用ウィンドウ内で「まだ使えない。」と表示する。

## v17追記: キャンプ内サブ画面

v17では、キャンプから以下の専用画面へ遷移する基盤を置く。

```text
キャンプ
  ├─ メンバーを見る → キャラクター詳細
  ├─ 隊列 → 隊列画面
  ├─ 装備確認 → 所持品概要
  ├─ 呪文 → メンバー選択 → 呪文回数確認
  ├─ アイテム → メンバー選択 → 所持品確認
  └─ 出る
```

この段階では、装備変更・呪文使用・アイテム使用は行わない。
まず、画面遷移と情報確認の骨格を固定する。

通常探索画面の `隊列` ボタンは、隊列専用ウィンドウへのショートカットとして扱ってよい。
ただし、隊列画面も専用ウィンドウであり、表示中は通常移動操作を止める。

## v18追記: 探索イベントオブジェクト

v18では、宝箱以外の探索イベントオブジェクトも専用ウィンドウで扱う。

対象:

- 祭壇
- 石像
- 床の紋様
- 罠検知時の床の印

画面遷移:

```text
通常探索画面
  ↓ 調べる
探索イベントウィンドウ
  ↓ 離れる
通常探索画面
```

専用ウィンドウ表示中は、通常移動・旋回・調べるを受け付けない。

## v29追加: 装備変更フロー

v29では、キャンプ内の装備変更を以下の最小フローで扱う。

```text
キャンプ
  ├─ 装備 → メンバー選択 → 所持品選択 → 装備 / 外す
  └─ アイテム → メンバー選択 → 所持品選択 → 使う / 装備 / 外す / 渡す / 捨てる
```

装備変更専用の複雑なスロット編集画面はまだ作らない。v28で作成した所持品選択画面を基準にし、装備変更も同じ操作文法で扱う。
