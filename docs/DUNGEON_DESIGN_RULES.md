# 3Dダンジョン設計上の約束事

この文書は、タイトル未定 3Dダンジョン試作における今後の設計判断を固定するためのものです。

## 1. 階段配置の基本方針

### 1.1 角配置は原則避ける

下り階段・上り階段は、原則としてマップの角には置かない。

理由:

- 角配置では、階段の開口部、側壁、進行方向、下階または上階との接続方向を自然に見せにくい。
- 下り階段と上り階段を対応させる際、下階側の着地点・向き・周囲の壁配置と矛盾しやすい。
- 一人称視点では、角にある階段が「床上の段差」や「台」に見えやすく、下層へ続く構造として認識されにくい。

### 1.2 階段は通路途中または階段室に置く

推奨配置は以下のどちらかとする。

- 直線通路の途中から1〜2マス余裕を持たせた位置
- 小さな踊り場を持つ階段室

階段室を使う場合は、最低限以下を確保する。

- 階段前に1マス以上の立ち位置
- 階段の左右に側壁または手すり相当の形状
- 下階または上階への接続方向が明確に分かる向き

## 2. 下り階段の見た目

下り階段は、床上に置いた段差オブジェクトとして扱わない。

必須要素:

- 現在階の床に開口部を作る
- 段は床面より下へ沈める
- 左右に側壁を付ける
- 下方向または奥方向に暗がりを置く
- 下層へ続く空間があるように見せる

B2F全体を実装する前でも、下層へ続く穴・暗がり・短い踊り場は視覚表現として用意する。

## 3. 上り階段との整合性

下り階段を配置するときは、将来の上り階段と対応できる前提で置く。

確認項目:

- 下り階段の降下方向と、下階の上り階段の向きが矛盾しないこと
- 下階に到着したとき、プレイヤーの向きが不自然にならないこと
- 上り階段から戻ったとき、元の階の位置・向きに戻れること
- 階段の周囲に最低限の踊り場または通路余白があること

## 4. 実装時の扱い

階層移動そのものが未実装の段階でも、階段の見た目は将来の階層接続を前提に作る。

禁止:

- 角に置いた床上オブジェクトだけで下り階段を表現すること
- 下層への穴や暗がりがないまま、階段マーカーだけを置くこと
- 下り階段と上り階段の向き・着地点を後から場当たり的に決めること

許容:

- B2F全体の実装前に、見た目用の短い下降空間だけを作ること
- 本実装前に、階段室だけを仮配置して視認性をテストすること

## 5. 今後の検査観点

階段を追加・修正した場合は、以下を確認する。

- 正面から見て下り階段と認識できるか
- 横から見ても床上の台に見えないか
- 階段マスに近づいたときに開口部が自然に見えるか
- 将来の下階マップと接続しても矛盾しない配置か
- 上り階段を同じルールで作図できるか
