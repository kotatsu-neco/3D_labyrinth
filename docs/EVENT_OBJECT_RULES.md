# イベントオブジェクト設計上の約束事

この文書は、固定一人称3Dダンジョンで扱う宝箱・石像・祭壇・碑文・罠床などのイベントオブジェクトに関する設計ルールを固定するためのものです。

## 1. 視点上下操作は追加しない

本プロトタイプでは、下を見る・上を見るといった視点上下操作は原則として追加しない。

理由:

- 操作体系が複雑になり、固定グリッド式ダンジョンRPGの手触りが崩れやすい。
- スマホ縦画面では、視点操作と移動操作が干渉しやすい。
- 小さな床置きオブジェクトを見るためだけに視点上下を入れると、設計全体の負担が増える。

## 2. 3D画面とイベントウィンドウの役割分担

### 2.1 3D画面の役割

3Dダンジョン画面は、イベント対象を「発見する」ための画面とする。

- 宝箱があることが分かる
- 祭壇や石像があることが分かる
- レバーや碑文があることが分かる
- 床罠や魔法陣の存在が分かる

ただし、細部操作や複数選択肢は3D画面内に詰め込まない。

### 2.2 イベントウィンドウの役割

イベントウィンドウは、対象を「操作する」ための画面とする。

宝箱では以下をイベントウィンドウで扱う。

- 調べる
- 担当者を選ぶ
- 必要な場合だけ罠を解除する
- 開ける
- 離れる

石像・祭壇・碑文も、読む・祈る・触れる・捧げる等の操作はイベントウィンドウで扱う方針とする。

## 3. 宝箱のルール

宝箱は、3D空間では発見用の簡易表示に留める。

- 通路中央を塞がない
- 壁際または小部屋内に置く
- 鍵穴・罠・中身などの詳細は3Dモデル上で表現しない
- 罠解除などの操作はイベントウィンドウで行う

## 4. 石像・祭壇・碑文のルール

石像・祭壇・碑文は、通常視野に入る高さと大きさで配置する。

推奨:

- 正面壁に置く
- 壁龕に置く
- 壁際の台座に載せる
- 胸から目線程度の高さに中心が来るようにする

避ける:

- 床に小さく直置きする
- 通路中央に置いて通行を邪魔する
- 見えない位置に置いて、調べる操作だけで反応させる

## 5. 床面イベントの扱い

床面に置いてよいものは、床全体の模様や色で認識できるものに限定する。

例:

- 罠床
- 魔法陣
- 血痕
- 床のひび
- 落とし穴
- 床スイッチ

小さな鍵・巻物・祭具などを床に直置きして、固定視点で見つけさせる設計は避ける。

## 6. 検査観点

イベントオブジェクトを追加・修正した場合は、以下を確認する。

- 通常視野で存在を認識できるか
- 通路を塞がないか
- 近づいたときに見切れすぎないか
- 「調べる」対象として納得できるか
- 詳細操作はイベントウィンドウ側に分離されているか

## 7. イベント文言の抑制

イベントウィンドウでは、プレイヤーが想像で補うべき情報を先回りして説明しない。

禁止:

- 危険、罠、不穏さを事前に示唆する説明文
- 「未確認」「未解除」など、UI側が先に状態を説明する表示
- 雰囲気づけのつもりで、攻略上のヒントになる文言
- 対象物の細部を過剰に説明する文章

宝箱の基本操作は、原則として以下の流れにする。

1. `調べる`
2. 担当者を選ぶ
3. 罠を発見した場合だけ `罠を解除する` を選択肢に追加
4. `罠を解除する`
5. 担当者を選ぶ
6. 解除成功または解除失敗

表示文は、原則として短くする。

例:

- `罠を発見した。`
- `解除した。`
- `解除に失敗した。`
- `開いた。`
- `開かなかった。`
