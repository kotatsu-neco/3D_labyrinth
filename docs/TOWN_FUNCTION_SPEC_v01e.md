# TOWN_FUNCTION_SPEC v01e

## 目的

この文書は、暗澹の石櫃 v34a を起点に、街機能をWizardry風RPGとして実装するための正本仕様である。Codex実装では本書を最優先し、旧版文書と矛盾する場合は本書を正とする。

## 用語定義

- **街フォーカス**: UI操作対象が街画面にある状態。冒険者が街へ帰還したことを意味しない。
- **帰還**: パーティーまたは冒険者が迷宮から街へ物理的に戻った状態変化。本書のv01実装では、キャンプからの帰還機能は作らない。
- **中断**: 迷宮内パーティーの位置・状態を保持したまま、操作フォーカスだけを街へ戻すこと。
- **休止中パーティー**: 中断により迷宮内に残っているパーティー。
- **遭難パーティー**: 全滅または全員行動不能により迷宮内に残っているパーティー。
- **救助隊**: 街で新たに編成し、迷宮内の遭難者を探すパーティー。
- **回収**: 救助隊が迷宮内で遭難者を発見し、空き枠を使って同行・搬送すること。

## 最重要原則

1. ダンジョン内で全滅したパーティーは自動で街へ帰還しない。
2. 全滅パーティーは迷宮内に残る。
3. 全滅後に街画面へ操作フォーカスを戻すことは許可する。ただし、冒険者・所持品・金・状態・位置を街へ移動してはならない。
4. 救助隊は街で新たに編成し、迷宮内キャンプの `周辺を探索する` によって遭難者を発見する。
5. `冒険を中断する` は帰還ではない。休止中パーティーは街施設の対象にならない。
6. 街の `冒険を再開する` は、迷宮内に休止しているパーティーを保存位置から再開する機能である。
7. パーティー最大人数は6人。救助対象を連れ帰るには空き枠が必要である。
8. 酒場はパーティー編成。訓練場は冒険者個人管理。両者を混同しない。
9. 宿屋は個人単位で宿泊し、レベルアップ判定を行う。
10. 寺院は街まで回収済みの対象だけを処置する。迷宮内の未回収者を自動処置しない。
11. 街UIはダンジョン内UIと同じ黒背景・白枠・モノスペース調の親和性を維持する。
12. 宿屋の支払いと寺院の支払いを混同しない。宿屋は宿泊対象本人の個人支払いでよいが、寺院では死亡・灰化・石化・麻痺など自力で支払えない対象を処置するため、必ず支払者を選択する。
13. 商店は個人単位の施設である。購入・売却・鑑定・解呪は、最初に対象キャラクターを選び、そのキャラクターの所持金・所持品枠・所持品だけを対象に処理する。

## v34a時点の前提

- 起動時は街画面。
- 街施設UIの器はある。
- 酒場/訓練場/宿屋/寺院/商店の本機能は未実装。
- キャンプには `周辺を探索する` と `冒険を中断する` がある。
- v34aでは休止中パーティーは1件のみ保持。
- 遭難記録は最小表示のみ。
- `PARTY_MEMBERS` は現在固定配列であり、複数パーティーや街所在管理には不十分。
- v34a既存コードには戦闘勝利時にレベルアップする処理があるため、街機能実装時に必ず宿屋レベルアップへ移管する。

## データモデル

### State

```js
{
  focus: "town" | "dungeon",
  adventurers: Adventurer[],
  parties: Party[],
  activePartyId: string | null,
  currentTownPartyId: string | null,
  suspendedParties: string[],
  strandedRecords: StrandedRecord[]
}
```

### 不変条件

- `focus === "dungeon"` のとき、`activePartyId` は `status === "ACTIVE"` のPartyを指す。
- `focus === "town"` のとき、`activePartyId` は原則 `null` とする。ただし実装移行中は参照互換のため保持してもよいが、街施設対象判定には使わない。
- 同時に `status === "ACTIVE"` のPartyは最大1件。
- 1人のAdventurerが複数Partyに所属してはならない。
- `Party.memberIds.length <= 6` を常に保証する。
- `location.type` とParty/StrandedRecordの所属は矛盾してはならない。

### Adventurer

```js
{
  id: string,
  name: string,
  race: string,
  alignment: "GOOD" | "NEUT" | "EVIL",
  className: string,
  level: number,
  xp: number,
  ageDays: number,
  hp: number,
  maxHp: number,
  ac: number,
  status: "OK" | "POISONED" | "ASLEEP" | "AFRAID" | "PARALYZED" | "STONED" | "OUT" | "DEAD" | "ASHES" | "LOST",
  stats: { str: number, iq: number, pie: number, vit: number, agi: number, luc: number },
  spells: { mage: number[], priest: number[] },
  knownSpells: { mage: string[], priest: string[] },
  gold: number,
  items: ItemInstance[],
  location: AdventurerLocation,
  carriedFromStrandedId?: string | null
}
```

### AdventurerLocation

```js
type AdventurerLocation =
  | { type: "TOWN" }
  | { type: "PARTY"; partyId: string }
  | { type: "SUSPENDED"; partyId: string }
  | { type: "STRANDED"; strandedId: string }
  | { type: "CARRIED"; partyId: string; strandedId: string }
  | { type: "RECOVERED" }
  | { type: "LOST" };
```

- `TOWN`: 酒場で通常加入できる。宿屋・寺院・訓練場の対象になりうる。
- `PARTY`: 街で編成中または迷宮探索中のPartyに所属している。
- `SUSPENDED`: 迷宮内で中断中のPartyに所属している。街施設の通常対象に出さない。
- `STRANDED`: 全滅/行動不能/遭難により迷宮内に残留している。街施設の通常対象に出さない。
- `CARRIED`: 迷宮内で救助隊が同行・搬送中。街施設の対象に出さない。
- `RECOVERED`: 街まで持ち帰られた回収済み対象。状態に応じて寺院・宿屋・訓練場の対象になりうる。
- `LOST`: 復活不能。通常操作対象から除外し、名簿では記録として表示する。

### Party

```js
{
  id: string,
  name: string,
  memberIds: string[],
  status: "IN_TOWN" | "ACTIVE" | "SUSPENDED" | "STRANDED" | "DISBANDED",
  floor: string | null,
  x: number | null,
  z: number | null,
  dir: number | null,
  objective: "NORMAL" | "RESCUE" | "RECOVERY" | null
}
```

### StrandedRecord

```js
{
  id: string,
  partyId: string,
  floor: string,
  x: number,
  z: number,
  dir: number,
  memberIds: string[],
  memberStatuses: { id: string, status: string, hp: number }[],
  discovered: boolean,
  carriedMemberIds: string[],
  recoveredMemberIds: string[],
  remainingMemberIds: string[],
  note: string
}
```

- `carriedMemberIds`: 迷宮内で救助隊が同行・搬送中の対象。
- `recoveredMemberIds`: 街まで持ち帰られた対象。
- `remainingMemberIds`: 迷宮内に残っている対象。

## 状態遷移

### 街で新規パーティーを作る

1. 酒場で `TOWN` の冒険者を選ぶ。
2. `Party.status = "IN_TOWN"` の街編成パーティーへ追加する。
3. 追加された冒険者は `location = { type: "PARTY", partyId }` になる。

### 迷宮へ出発する

1. `currentTownPartyId` が存在し、メンバーが1人以上であること。
2. Partyを `IN_TOWN` から `ACTIVE` へ変更する。
3. `activePartyId = partyId`、`focus = "dungeon"` とする。
4. 初期位置または指定位置へ配置する。

### 冒険を中断する

1. キャンプから `冒険を中断する` を選ぶ。
2. `Party.status = "SUSPENDED"` にする。
3. Partyの `floor/x/z/dir` を保存する。
4. メンバー全員を `location = { type: "SUSPENDED", partyId }` にする。
5. `activePartyId = null`、`focus = "town"` にする。
6. HP、状態異常、呪文回数、所持品、金、経験値は一切変更しない。
7. 実行UIに「帰還」と表示してはならない。

### 冒険を再開する

1. 街の `冒険を再開する` で `SUSPENDED` Partyを選ぶ。
2. Partyを `ACTIVE` に戻す。
3. メンバー全員を `location = { type: "PARTY", partyId }` に戻す。
4. 保存された `floor/x/z/dir` を復元する。
5. `activePartyId = partyId`、`focus = "dungeon"` とする。
6. `suspendedParties` から該当Partyを除外する。

### 全滅/全員行動不能

1. `ACTIVE` Partyの全メンバーがダンジョン行動不能になった場合に発生する。
2. Partyを `STRANDED` にする。
3. `StrandedRecord` を作成する。
4. メンバー全員を `location = { type: "STRANDED", strandedId }` にする。
5. HP、状態、所持品、金、経験値、位置は迷宮内に残る。
6. `activePartyId = null` とする。
7. UI上、操作フォーカスを街へ戻してよい。この場合も「帰還」「街へ戻った」「救出扱い」と表示してはならない。

### 救助隊による発見・搬送

1. 救助隊が迷宮内キャンプで `周辺を探索する` を選ぶ。
2. 同一階層かつマンハッタン距離1以内の `StrandedRecord` を確定発見する。
3. v01では発見率、職業補正、失敗判定、ランダム遭遇は実装しない。
4. 発見対象から連れ帰る対象を選ぶ。
5. 空き枠数 = `6 - activeParty.memberIds.length`。
6. 選択可能人数は空き枠数以下。
7. DEAD / ASHES / STONED / PARALYZED / OK のいずれでも、1人につき1枠を占有する。
8. 選択した対象を救助隊Partyへ追加し、対象の `location = { type: "CARRIED", partyId: activeParty.id, strandedId }` とする。
9. `StrandedRecord.carriedMemberIds` へ追加し、`remainingMemberIds` から除外する。
10. 6人満員の場合、誰も連れ帰れない。
11. v01では荷物だけの回収は実装しない。

### 街への物理帰還

v01ではキャンプからの帰還手段は実装しない。ただし将来の帰還処理に備え、以下の規則を固定する。

1. `CARRIED` の冒険者がいるPartyが街へ物理帰還したとき、該当冒険者は `location = { type: "RECOVERED" }` になる。
2. `RECOVERED` になった冒険者は状態に応じて寺院・宿屋・訓練場の対象になりうる。
3. `CARRIED` のままでは街施設の対象に出してはならない。

## 街トップ

### 表示要素

- 街名/バージョン/HUD。
- 現在編成中パーティー概要。
- 街にいる冒険者数。
- 休止中パーティー数。
- 遭難記録数。
- 施設ボタン。

### 施設

- 酒場
- 訓練場
- 宿屋
- 寺院
- 商店
- 冒険を再開する
- 遭難記録
- 迷宮へ

### 迷宮へ

- `IN_TOWN` Partyが1人以上いる場合のみ出発可能。
- パーティーが6人かつ遭難記録が残っている場合、「救助対象を連れ帰る空き枠がありません」と警告する。ただし通常探索として出発は可能。
- 休止中パーティーを再開する場合は `冒険を再開する` を使う。

## 酒場

### 役割

酒場はパーティー編成施設である。新規作成・転職・削除は扱わない。

### 機能

1. 現在パーティーを見る。
2. 街にいる冒険者を加える。
3. パーティーから外す。
4. 隊列/現状態を見る。
5. 金を分配する。
6. パーティーを解散する。
7. 街へ戻る。

### 加入候補

加入候補は以下をすべて満たす冒険者に限る。

- `location.type === "TOWN"`
- `status !== "LOST"`
- 現在パーティーに所属していない。
- パーティー人数が6人未満。
- 属性互換性を満たす。

### 属性互換性

- GOOD と EVIL は同一パーティーに共存不可。
- NEUT は GOOD / EVIL のどちらとも組める。
- パーティー内にGOODがいる場合、EVILは加入不可。
- パーティー内にEVILがいる場合、GOODは加入不可。
- パーティー内がNEUTのみ、または空の場合、GOOD/NEUT/EVILのいずれも加入可能。

### 外す

- `IN_TOWN` Partyから外した冒険者は `location = { type: "TOWN" }` に戻る。
- `ACTIVE` / `SUSPENDED` / `STRANDED` Partyのメンバーは酒場から外せない。

### 金を分配する

- 現在の `IN_TOWN` Party全員の所持金を合算し、均等に再配分する。
- `base = Math.floor(totalGold / memberCount)`。
- `remainder = totalGold % memberCount`。
- 隊列順に、先頭から `remainder` 人へ1Gずつ追加する。
- パーティー共有金庫は作らない。

### パーティーを解散する

- 対象は `IN_TOWN` Partyのみ。
- 全メンバーを `TOWN` に戻す。
- Partyを `DISBANDED` にする。
- `currentTownPartyId = null` にする。
- `ACTIVE` / `SUSPENDED` / `STRANDED` Partyは解散できない。

## 訓練場

### 役割

訓練場は冒険者個人管理施設である。パーティー編成は扱わない。

### 入場時のパーティー解散

- 訓練場に入るとき、`IN_TOWN` Partyが存在する場合は確認を出す。
- ユーザーが承認した場合、街の編成中パーティーを解散する。
- 解散処理は酒場の「パーティーを解散する」と同じ。
- `ACTIVE` / `SUSPENDED` / `STRANDED` Partyは解散しない。

### 機能

1. 新規冒険者を作成する。
2. 冒険者名簿を見る。
3. 個人詳細を見る。
4. 転職する。
5. 引退/削除する。
6. 街へ戻る。

### 新規作成 v01範囲

v01では以下の最小実装に固定する。

1. 名前はプリセット候補から選ぶ。任意入力UIは後続実装。
2. 種族を選ぶ。
3. 属性を選ぶ。
4. 職業を選ぶ。
5. 能力値は種族基本値 + 職業別固定補正で生成する。
6. ボーナスポイント配分UIは実装しない。
7. 作成後、`location = { type: "TOWN" }` として登録する。

データ構造は将来の任意名入力・ボーナスポイント配分に耐える形にする。

### 名簿

名簿には以下を表示する。

- 名前
- 種族
- 職業
- レベル
- 状態
- 所在
  - 街
  - 編成中
  - 迷宮内休止
  - 遭難/未回収
  - 搬送中
  - 回収済み
  - LOST

### 転職

#### 条件

- 対象は `TOWN` または `RECOVERED` の冒険者。
- `LOST` は不可。
- `SUSPENDED` / `STRANDED` / `CARRIED` は不可。
- 新職業の能力値・属性条件を満たすこと。

#### 副作用

- 新職業のレベル1になる。
- 経験値は0になる。
- 能力値は種族最低値まで下がる。
- 装備はすべて外れる。
- 所持品は保持する。
- 新職業で装備不能な装備は未装備状態で残る。
- 既得呪文知識は保持する。
- 呪文使用回数は新職業/既得呪文に基づいて再計算する。
- `ageDays += selectedWeeks * 7` を行う。

#### 加齢

`selectedWeeks` は以下から等確率で1つ選ぶ。

```text
252 / 304 / 356
```

### 引退/削除

- 2段階確認を必須にする。
- `SUSPENDED` / `STRANDED` / `CARRIED` の冒険者は削除不可。
- `LOST` はv01では削除不可。名簿に記録として残す。

## 宿屋

### 役割

宿屋は、街にいる冒険者を個人単位で宿泊させ、HP回復・呪文回復・レベルアップ判定を行う施設である。

### 対象

- `location.type === "TOWN" || location.type === "RECOVERED"`。
- `status === "OK" || status === "POISONED" || status === "AFRAID" || status === "ASLEEP" || status === "OUT"`。
- `DEAD` / `ASHES` / `LOST` / `STONED` / `PARALYZED` は宿泊不可。先に寺院処置が必要。

### POISONEDの扱い

- 宿屋は `POISONED` を治療しない。
- v01では街滞在中の毒継続ダメージは発生させない。
- `POISONED` を治療する街機能は後続実装とする。

### 部屋ランク

| ID | 表示名 | 原典対応 | HP回復量/週 | 料金/週 | 経過 |
|---|---|---|---:|---:|---:|
| stable | 馬小屋 | Stables | 0 | 0 | 1週固定 |
| barracks | 大部屋 | Barracks | 1 | 10 | 必要週数 |
| double | 二人部屋 | Double Occupancy | 3 | 50 | 必要週数 |
| private | 個室 | Private Room | 7 | 200 | 必要週数 |
| royal | 王室 | Royal Suite | 10 | 500 | 必要週数 |

### 宿泊週数

```js
function innWeeksToStay(member, room) {
  if (room.id === "stable") return 1;
  const missingHp = Math.max(0, member.maxHp - member.hp);
  return Math.max(1, Math.ceil(missingHp / room.hpPerWeek));
}
```

### 支払い

- v01では宿泊対象本人の所持金からのみ支払う。
- パーティープール支払いは実装しない。
- 料金が0の部屋は支払判定なしで宿泊できる。
- 料金が不足した場合、不足する週の処理は行わず、その時点で宿泊を終了する。

### 宿泊処理

1. 宿泊対象を1人選ぶ。
2. 部屋を選ぶ。
3. `weeks = innWeeksToStay(member, room)` を計算する。
4. `processedWeeks = 0` とする。
5. 各週について、料金を支払える場合だけ支払い・回復・加齢を行う。
6. 各週ごとに `ageDays += 7`。
7. HP回復は `hp = min(maxHp, hp + room.hpPerWeek)`。馬小屋は0回復。
8. 1週以上処理できた場合、宿泊後に呪文使用回数を最大値へ戻す。
9. 1週以上処理できた場合、宿泊後にレベルアップ判定を行う。
10. 一度の宿泊で上昇するレベルは最大1。

### レベルアップ

- レベルアップは宿屋でのみ発生する。
- 戦闘勝利時にレベルアップしてはならない。
- 戦闘勝利時はEXP/GOLD配分のみ行う。
- 既存コードに戦闘勝利時レベルアップ呼び出しがある場合、削除または無効化する。
- 経験値が必要値以上なら、宿泊後に1レベルだけ上昇する。
- 複数レベル分の経験値がある場合、もう一度宿泊すれば次のレベルアップ判定が行われる。
- レベルアップ時、最大HP上昇、呪文習得、呪文使用回数更新を行う。
- 能力値変動はv01では既存簡易ロジックまたは固定最小変動でよい。ただし、年齢による能力値低下リスクを後続実装できる関数境界を作る。

### 年齢

- 内部値は `ageDays`。
- 表示は `Math.floor(ageDays / 365)` 歳を基本とする。
- 詳細画面では `Math.floor((ageDays % 365) / 7)` 週を併記してよい。
- 年齢による能力値低下・老衰はv01では実装しない。関数境界だけ残す。

## 寺院

### 役割

寺院は、街にいる、または街へ回収済みの麻痺・石化・死亡・灰化した冒険者を処置する施設である。迷宮内未回収者は対象外。

寺院は、処置対象本人が支払う施設ではない。寺院では、処置対象を選び、処置内容を確認し、支払者を選んでから処置を実行する。必要に応じて、現在街パーティー内の金を1人に集める。

### 対象

- `location.type === "TOWN" || location.type === "RECOVERED"`。
- `status` が `PARALYZED` / `STONED` / `DEAD` / `ASHES` のいずれか。
- `LOST` は復活不可。
- `STRANDED` / `SUSPENDED` / `CARRIED` は処置不可。

### 処置表

| 状態 | 処置 | 成功率 | 費用 | 成功時 | 失敗時 | 加齢 |
|---|---|---:|---:|---|---|---:|
| PARALYZED | 麻痺治療 | 100% | 100 * LV | OK | なし | 1d52週 |
| STONED | 石化解除 | 100% | 200 * LV | OK | なし | 1d52週 |
| DEAD | 蘇生 | clamp(0,100,50 + 3 * VIT)% | 250 * LV | OK / HP1 | ASHES | 1d52週 |
| ASHES | 灰から蘇生 | clamp(0,100,40 + 3 * VIT)% | 500 * LV | OK / HP全快 | LOST | 1d52週 |
| LOST | なし | 0% | - | 不可 | 不可 | - |

### 成功判定

```js
function clamp(min, max, value) {
  return Math.max(min, Math.min(max, value));
}

function templeSuccessPercent(member, service) {
  const vit = Number(member.stats?.vit || 0);
  if (service === "raiseDead") return clamp(0, 100, 50 + 3 * vit);
  if (service === "raiseAshes") return clamp(0, 100, 40 + 3 * vit);
  return 100;
}

function templeAgeCostDays() {
  return rollRange(1, 52) * 7;
}
```

- `rollRange(1, 52)` は1以上52以下の整数を等確率で返す。
- `rollRange(1, 100) <= successPercent` なら成功。
- 成功率100%の処置はロールせず成功としてよい。

### 支払い

寺院の支払いは宿屋と同じ扱いにしてはならない。寺院の処置対象は `PARALYZED` / `STONED` / `DEAD` / `ASHES` であり、本人が自力で支払い行為を行うと解釈してはいけない。

#### 支払者

- 寺院では、処置対象とは別に `支払者` を必ず選択する。
- 支払者候補は、現在街にいる `IN_TOWN` Party のメンバーに限定する。
- 支払者候補は `location.type === "TOWN"` を満たすこと。
- 支払者候補の状態は `OK` または `POISONED` に限定する。
- `PARALYZED` / `STONED` / `DEAD` / `ASHES` / `LOST` の冒険者は支払者候補に出さない。
- 支払者候補が存在しない場合、寺院処置は実行できない。
- 支払者の所持金が費用未満の場合、その支払者では処置を実行できない。
- 処置対象本人の `gold` を直接減算して支払い済みとする処理は禁止する。ただし、下記の「お金を集める」によって、現在街パーティー内の金を支払者へ集約した後、その支払者が支払うことはできる。

#### お金を集める

寺院には、処置費用を用意するための `お金を集める` を置く。

処理手順:

1. 寺院画面で `お金を集める` を選ぶ。
2. 受取先、すなわち金を集める相手を1人選ぶ。
3. 受取先候補は、現在街にいる `IN_TOWN` Party のうち、支払者候補と同じ条件を満たす冒険者に限定する。
4. 現在街にいる `IN_TOWN` Party 全員の `gold` を合算する。
5. 合算した全額を受取先の `gold` に設定する。
6. 受取先以外の同Partyメンバーの `gold` は0にする。
7. `IN_TOWN` Party 以外、休止中パーティー、遭難中パーティー、迷宮内の未回収者、`CARRIED`、`RECOVERED`、`LOST` の金は集約対象に含めない。

この処理は酒場の `金を分配する` とは別機能である。

- 酒場の `金を分配する`: 現在街パーティー内の金を均等配分する。
- 寺院の `お金を集める`: 現在街パーティー内の金を、選択した1人へ集中する。

#### 処置時の支払い

- 処置実行時は、選択した支払者の `gold` から費用を減算する。
- 処置を実行した場合、成功/失敗に関わらず費用を消費する。
- 処置を実行した場合、成功/失敗に関わらず処置対象の `ageDays += templeAgeCostDays()` を行う。
- 支払者の `ageDays` は寺院処置では増加しない。

### 成功/失敗処理

- `PARALYZED` 成功: `status = "OK"`。
- `STONED` 成功: `status = "OK"`。
- `DEAD` 成功: `status = "OK"`, `hp = 1`。
- `DEAD` 失敗: `status = "ASHES"`, `hp = 0`。
- `ASHES` 成功: `status = "OK"`, `hp = maxHp`。
- `ASHES` 失敗: `status = "LOST"`, `location = { type: "LOST" }`, `hp = 0`。

## 商店

### 役割

商店は、冒険者個人が購入・売却・鑑定・解呪を行う施設である。パーティー全体で一括購入、一括売却、一括鑑定、一括解呪を行う施設ではない。

商店に入った後、購入・売却・鑑定・解呪の各操作では、最初に対象キャラクターを選択する。以後の処理は、選択した対象キャラクターの `gold`、`inventory`、`equipment/equippedSlot`、鑑定状態、呪い状態だけに対して行う。

### 対象キャラクター

- 対象候補は、現在街にいる `IN_TOWN` Party のメンバーに限定する。
- 対象候補は `location.type === "TOWN"` を満たすこと。
- 対象候補は `LOST` ではないこと。
- `DEAD` / `ASHES` / `STONED` / `PARALYZED` は通常の買い物・売却・鑑定・解呪対象に出さない。
- v01では、商店対象候補は `OK` または `POISONED` の冒険者に限定する。
- 迷宮内で休止中、遭難中、未回収、`CARRIED`、`RECOVERED` だが街パーティー外の冒険者は対象候補に出さない。

### 購入

- 購入品は、選択した対象キャラクターの所持品へ追加する。
- 支払いは、選択した対象キャラクター本人の `gold` から行う。
- 対象キャラクターの `gold` が価格未満なら購入不可。
- 対象キャラクターの所持品数が10個以上なら購入不可。
- 購入時に他メンバーの `gold` を自動的に合算・借用・移動してはならない。
- 購入後、店在庫を減らすかどうかは在庫仕様に従う。v01では在庫管理は未実装なので、購入機能自体も後続実装とする。

### 売却

- 売却対象は、選択した対象キャラクターの所持品に限定する。
- 他メンバーの所持品を同じ画面で混在表示してはならない。
- 売却代金は、選択した対象キャラクター本人の `gold` に加算する。
- 装備中アイテムを売却する場合は、売却前に装備解除するか、装備中のため売却不可にする。どちらを採用するかは実装前に明記する。v01では売却機能は後続実装なので未確定のまま実装しない。
- 呪われた装備品は、解呪前に売却できない仕様を将来前提とする。

### 鑑定

- 鑑定対象は、選択した対象キャラクターの未鑑定所持品に限定する。
- 鑑定費用は、選択した対象キャラクター本人の `gold` から支払う。
- 鑑定成功後、対象アイテムの鑑定状態を更新する。
- Bishopによる鑑定は商店鑑定とは別仕様として後続実装候補に留める。

### 解呪

- 解呪対象は、選択した対象キャラクターの呪われた所持品または装備品に限定する。
- 解呪費用は、選択した対象キャラクター本人の `gold` から支払う。
- 解呪成功後、対象アイテムの呪い状態を解除し、必要なら装備解除可能にする。

### 商店と金操作

商店には、購入・鑑定・解呪の支払いに備えるための `お金を集める` を置く。

`お金を集める` は、現在街にいる `IN_TOWN` Party 全員の `gold` を、選択した1人へ集中させる機能である。これは商店での買い物をパーティー一括処理にするものではない。

処理手順:

1. 商店トップで `お金を集める` を選ぶ。
2. 受取先キャラクターを選択する。
3. 対象範囲は、現在街にいる `IN_TOWN` Party のメンバーだけとする。
4. 迷宮内休止中、遭難中、未回収、`CARRIED`、`RECOVERED` だが現在街パーティー外、`LOST` の冒険者の金は対象にしない。
5. 対象メンバー全員の `gold` を合算する。
6. 受取先の `gold` を合算額にする。
7. 受取先以外の対象メンバーの `gold` を0にする。
8. 処理結果として、誰にいくら集まったかを表示する。

金操作の区別:

- 酒場の `金を分配する`: 現在街パーティー内の金を均等配分する。
- 寺院の `お金を集める`: 寺院支払いに備えて、現在街パーティー内の金を選択した1人へ集中させる。
- 商店の `お金を集める`: 商店で買い物・鑑定・解呪を行う担当者に、現在街パーティー内の金を集中させる。

商店の `お金を集める` 後も、購入・鑑定・解呪の支払いは、選択した対象キャラクター本人の `gold` から行う。購入品は選択した対象キャラクター本人の `inventory` へ入る。鑑定・解呪対象も選択した対象キャラクター本人の所持品に限定する。

商店が、パーティー全体の財布、共有金庫、暗黙の金プールを支払い時に参照する処理は禁止する。支払い前に金を動かす必要がある場合は、明示的に `お金を集める` を実行させる。

### v01範囲

街機能の初回Codex実装では、商店はUIの器を維持する。購入・売却・鑑定・解呪の本処理は後続実装。ただし、UIの器は将来の個人単位処理に合わせ、以下の構造を前提にする。

1. 商店トップ。
2. 操作選択: `買う` / `売る` / `鑑定` / `解呪` / `お金を集める` / `街へ戻る`。
3. `買う` / `売る` / `鑑定` / `解呪` は、各操作ごとに対象キャラクターを選択。
4. 対象キャラクターの所持金・所持品枠・所持品だけを表示。
5. `お金を集める` は受取先キャラクターを選択し、現在街パーティー内の金をその1人へ集中する。

以下は後続実装。

- 購入
- 売却
- 鑑定
- 解呪
- 在庫管理
- 未鑑定アイテム
- 呪い

### 将来前提

- 1人の所持品上限は10個。
- 未鑑定品は装備リスクがある。
- 呪われた装備は解呪しないと外せない。
- Bishopによる鑑定は後続実装候補。

## 冒険を再開する

### 役割

迷宮内で中断中のパーティーを保存位置から再開する。

### 表示

- パーティー名またはメンバー名。
- 階層。
- 座標。
- 方角。
- メンバー状態。

### 処理

「状態遷移 > 冒険を再開する」に従う。

## 遭難記録

### 役割

迷宮内に残された全滅/行動不能パーティーを確認する。

### 表示

- 階層。
- 座標。
- メンバー名。
- 各メンバーの状態。
- 発見済み/未発見。
- 搬送中/回収済み/未回収。

### 禁止

- 街画面から自動救出しない。
- 街画面から自動治療しない。
- 街画面から未回収キャラを酒場へ戻さない。

## キャンプ: 周辺を探索する

### 役割

救助隊が迷宮内で周辺を探索し、全滅パーティー・別冒険者・遺体・灰・石化者を発見する。

### v01発見判定

- 同一階層。
- 現在座標からマンハッタン距離1以内。
- 上記を満たす `StrandedRecord` は確定発見。
- v01では職業補正、失敗判定、ランダム遭遇、探索リスクは実装しない。

### 発見後

「状態遷移 > 救助隊による発見・搬送」に従う。

## キャンプ: 冒険を中断する

### 役割

迷宮内パーティーの位置・状態を保持し、操作フォーカスを街へ戻す。

### 禁止

- HPを回復しない。
- 状態異常を回復しない。
- 街に帰還した扱いにしない。
- 酒場の加入候補に戻さない。

## UI方針

- ダンジョン内UIと同じ黒背景、白枠、モノスペース調を維持する。
- 施設ごとに派手な色や現代的カードUIへ変えない。
- スマホ縦画面で、ボタンは押しやすくする。
- ステータス詳細画面とフォントサイズ感を合わせる。
- 「帰還」「救出扱い」「自動で街へ戻る」など、仕様誤解を招く文言を実行UIに出さない。
- `冒険を中断する` と `冒険を再開する` は、帰還ではないことを説明文で明示する。

## 受け入れ条件

- 全滅時にパーティー実体が街へ自動復帰しない。
- 全滅後、操作フォーカスが街へ戻る場合でも、冒険者の `location` は `STRANDED` である。
- 全滅したパーティーが遭難記録へ残る。
- 遭難者は未回収のままでは酒場・宿屋・寺院の通常対象に出ない。
- キャンプの `周辺を探索する` から遭難者を発見する導線がある。
- 6人満員の救助隊では救助対象を連れ帰れない。
- 救助対象は状態を問わず1人1枠を占有する。
- 酒場と訓練場の責務が分離されている。
- 訓練場に入ると、街の編成中パーティーが解散する。
- 宿屋は個人単位で宿泊する。
- 宿屋の部屋ランクがある。
- 宿泊後にレベルアップ判定がある。
- 戦闘勝利時にレベルアップしない。
- 宿泊・転職・寺院サービスが `ageDays` に反映される。
- DEAD蘇生失敗でASHES、ASHES蘇生失敗でLOSTになる。
- LOSTは復活不能。
