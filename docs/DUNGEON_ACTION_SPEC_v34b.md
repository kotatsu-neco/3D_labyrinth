# DUNGEON_ACTION_SPEC v34b

## 位置づけ

v34bは、v34aの街UI器を、確定した街機能仕様 `town_spec_v01e` に合わせて修正するバージョンである。
街機能の本実装ではない。Codex実装前に、UI設計がWizardry型の施設役割と衝突しないようにする。

## 最重要前提

- 全滅パーティーは街へ自動帰還しない。
- キャンプの `冒険を中断する` は帰還ではない。
- 街の `冒険を再開する` は、迷宮内に休止しているパーティーを保存位置から再開する導線である。
- 酒場はパーティー編成施設。
- 訓練場は冒険者個人管理施設。
- 宿屋は個人単位の宿泊施設。
- 寺院は処置対象と支払者を分ける施設。
- 商店は個人単位の買い物・売却・鑑定・解呪施設。
- 寺院と商店には `お金を集める` を置く。
- 酒場の `金を分配する` は均等配分、寺院/商店の `お金を集める` は選択した1人への金集中である。

## 街トップ

v34bの街トップは以下を表示する。

- 酒場
- 訓練場
- 宿屋
- 寺院
- 商店
- 冒険を再開する
- 遭難記録
- 迷宮へ

注記では、街が操作フォーカスの拠点であり、全滅パーティーを自動回収しないことを明示する。
また、宿屋・商店は個人単位、寺院は処置対象と支払者を分けることを明示する。

## 宿屋UI

v34bでは、宿屋は以下の画面構造を持つ。

1. 宿屋トップ
2. 宿泊対象キャラクター選択
3. 部屋ランク選択

部屋ランクは以下を表示する。

| 表示名 | HP回復 | 料金 | 経過 |
|---|---:|---:|---:|
| 馬小屋 | HP 0/週 | 0G | 1週固定 |
| 大部屋 | HP 1/週 | 10G/週 | 必要週数 |
| 二人部屋 | HP 3/週 | 50G/週 | 必要週数 |
| 個室 | HP 7/週 | 200G/週 | 必要週数 |
| 王室 | HP 10/週 | 500G/週 | 必要週数 |

v34bでは実処理は行わない。
後続実装では、宿泊対象本人の所持金から支払い、1週以上宿泊した場合のみ呪文回復とレベルアップ判定を行う。

## 寺院UI

v34bでは、寺院は以下の画面構造を持つ。

1. 寺院トップ
2. 処置対象選択
3. 支払者選択
4. お金を集める

寺院では、処置対象本人の所持金から直接支払ってはならない。
支払いは、別に選択した支払者の所持金から行う。
`お金を集める` は、現在街パーティーの金を選択した1人へ集中させる機能である。

v34bでは実処理は行わない。
後続実装では、未回収者・休止中・遭難中・CARRIED状態の者を寺院対象に出してはならない。
寺院サービスの加齢は `1d52週` 固定とする。

## 商店UI

v34bでは、商店は以下の操作を表示する。

- 買う
- 売る
- 鑑定
- 解呪
- お金を集める
- 街へ戻る

`買う` / `売る` / `鑑定` / `解呪` は、まず対象キャラクター選択画面へ進む。
商店は個人単位の施設であり、暗黙の共有財布やパーティー一括処理を行わない。

`お金を集める` は、買い物・鑑定・解呪を担当する1人へ金を集中させる補助機能である。
`お金を集める` 実行後も、購入・売却・鑑定・解呪は対象キャラクター本人の所持金・所持品だけを対象にする。

v34bでは実処理は行わない。

## 酒場UI

酒場は引き続き以下の器を持つ。

- 現在パーティー
- 街にいる冒険者
- 加える
- 外す
- 金を分配する
- 街へ戻る

`金を分配する` は、後続実装では現在街パーティー内の金を均等配分する。

## 訓練場UI

訓練場は引き続き以下の器を持つ。

- 新規作成
- 冒険者名簿
- 個人詳細
- 転職
- 引退/削除
- 街へ戻る

後続実装では、訓練場に入る際に、街で編成中のパーティーを解散する確認を行う。

## v34bで実装しないこと

- 宿屋・寺院・商店の実処理。
- 金の実移動。
- レベルアップ処理の宿屋移管。
- 訓練場入場時のパーティー解散確認実処理。
- 複数パーティー状態モデルへの完全移行。
