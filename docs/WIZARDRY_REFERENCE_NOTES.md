# Wizardry 参考調査メモ

この文書は、本作の画面設計・キャラクター詳細・キャンプ・宝箱・戦闘基礎を設計する前提として、Wizardry I〜III系の仕様を調査し、本作に採用する方針を整理するための参照メモです。

実装仕様そのものではありません。実装仕様は、必要に応じて `docs/SCREEN_FLOW_RULES.md`、`docs/COMBAT_RULES.md`、`docs/UI_STYLE_RULES.md` へ反映します。

## 1. 調査時の注意

Wizardry系は機種・移植・日本語版・攻略サイト・マニュアルにより表記やUIが異なる可能性があります。

以下を分けて扱います。

- 確認済みの大枠仕様
- 本作へ採用する仕様
- FC版1/2/3の実画面で追加確認すべき仕様
- 本作ではそのまま採用しない仕様

## 2. パーティと基本構造

Wizardry I系は、6人パーティを基本とします。

大枠として、以下の要素を持ちます。

- 種族
- アライメント
- 職業
- 上級職
- HP
- AC
- 状態
- 能力値
- Mage / Priest 系の呪文使用回数
- 所持品・装備

本作でも、6人パーティを基本にします。

## 3. 能力値

Wizardry I系の能力値は、以下の6種を基準にします。

- Strength
- I.Q.
- Piety
- Vitality
- Agility
- Luck

本作での表示方針:

- 詳細画面では、原典感を残すため `Strength` / `I.Q.` / `Piety` / `Vitality` / `Agility` / `Luck` または略称を併記する。
- 狭いUIでは、`STR` / `IQ` / `PIE` / `VIT` / `AGI` / `LUC` を使う。
- 日本語説明が必要な場合は、詳細ヘルプや説明画面で扱う。

対応:

```text
STR = Strength = 力
IQ  = I.Q.     = 知恵・知性
PIE = Piety    = 信仰心
VIT = Vitality = 生命力
AGI = Agility  = 敏捷性・素早さ
LUC = Luck     = 運
```

## 4. AC

ACは、攻撃の当たりにくさを表す値とします。

- 低いほどよい。
- 命中判定に使う。
- 命中後のダメージ軽減値ではない。
- プレイヤーキャラクターだけでなく、モンスターにもACを設定する。

詳細は `docs/COMBAT_RULES.md` を正本とします。

## 5. 呪文使用回数

MP制は採用しません。

Mage / Priest の2系統、各7レベルの使用回数制を基本とします。

表示例:

```text
MAGE   5/3/2/0/0/0/0
PRIEST 2/1/0/0/0/0/0
```

意味:

```text
1レベル呪文を5回使える
2レベル呪文を3回使える
3レベル呪文を2回使える
以降は0回
```

本作での方針:

- 通常探索画面のパーティカードには、呪文使用回数を表示しない。
- 通常探索画面には `MP` / `MAGIC` を出さない。
- 呪文使用回数は、キャラクター詳細、魔法、キャンプ内の専用ウィンドウで表示する。
- ビショップや転職後キャラクターは、Mage / Priest の両方を持ちうる。

## 6. 通常探索画面

本作での方針:

- 通常探索画面は、3D表示・パーティカード・基本操作に絞る。
- 常設の大きなメッセージ枠は置かない。
- パーティカードには以下を表示する。
  - 名前
  - 職業
  - HP
  - AC
  - 状態
- MPや呪文使用回数は通常カードに表示しない。

## 7. キャンプ画面

Campは、迷宮内で開く管理画面として扱います。

調査上の整理:

- Camp中の `INSPECT` は、パーティメンバーの詳細確認・キャラクター操作に関係する。
- Maze中の `Inspect` は、現在地点の調査、迷宮内に残された冒険者・死亡者・他パーティの探索に関係する可能性がある。
- この2つは同じ語でも役割が異なるため、本作では混同しない。

本作での方針:

```text
通常探索中の調べる
  = 周辺・扉・壁・イベント・遺体・他パーティなどを調べる行為

キャンプ内のキャラクター確認
  = パーティメンバーの詳細を見る行為
```

キャンプ画面の基本候補:

```text
キャンプ
  - メンバーを見る
  - 隊列
  - 全員装備
  - 呪文
  - アイテム
  - 出る
```

## 8. キャラクター詳細画面

キャラクター詳細画面は、単なる能力値表示ではなく、将来的にはキャンプ内操作やキャラクター操作の入口になります。

表示候補:

```text
名前 / LEVEL
ALIGN / RACE / CLASS

STR / I.Q. / PIE / VIT / AGI / LUC
HP / MAX HP
AC
STATUS

MAGE   0/0/0/0/0/0/0
PRIEST 0/0/0/0/0/0/0

ITEMS
```

v16での実装範囲:

- 通常探索カードからキャラクター詳細を開く。
- キャンプ画面の入口を作る。
- キャンプからキャラクターを選び、同じ詳細ウィンドウを開く。
- 詳細画面では閲覧のみ。

v16ではまだ実装しないもの:

- 装備変更
- アイテム使用
- アイテム受け渡し
- 呪文使用
- 識別
- 捨てる
- ゴールド集約

## 9. 宝箱と罠

本作での方針:

- v15cまでの宝箱イベントウィンドウ方式を維持する。
- `調べる → 誰が → 罠を発見 → 罠を解除する追加 → 誰が → 解除成功/失敗` の流れは維持する。
- 将来的にCALFO相当の魔法確認を追加する余地を残す。
- 宝箱の説明文や危険の匂わせは増やさない。
- 操作結果と事実は短文で表示する。

## 10. モンスター

モンスターにもACを設定します。

最低限必要な戦闘用パラメータ候補:

- 名前
- HP
- AC
- 攻撃力
- 命中力
- 攻撃回数
- 状態耐性
- 特殊能力
- 経験値
- ドロップ

本作では、プレイヤーの攻撃も対象モンスターのACを参照して命中判定します。

## 11. v16計画への反映

v16は、当初予定していた「キャラクター詳細だけ」ではなく、キャンプ画面との接続を含めた最小実装にします。

```text
通常探索画面
  ↓
キャンプ
  ├─ メンバーを見る
  │    └─ キャラクター詳細
  ├─ 隊列（未実装表示）
  ├─ 全員装備（未実装表示）
  ├─ 呪文（未実装表示）
  ├─ アイテム（未実装表示）
  └─ 出る

通常探索画面のキャラクターカード
  ↓
キャラクター詳細
```

この段階では、表示と画面遷移の骨格を作ります。
実際の装備変更・魔法使用・アイテム管理はv17以降へ回します。

## 12. 追加確認が必要なこと

- FC版1/2/3のキャンプ画面の正確な日本語メニュー表記
- FC版1/2/3のステータス詳細画面のレイアウト
- `INSPECT` の日本語訳と場面ごとの差異
- 能力値の日本語表記
- 呪文使用回数の日本語画面上の表記
- 宝箱画面の日本語コマンド表記

## 13. 禁止事項

- Camp内のキャラクター確認と、Maze中の地点調査を混同しない。
- MP制の表示を復活させない。
- 通常探索カードに呪文使用回数の合計値を表示しない。
- ACをダメージ軽減値として扱わない。
- Wizardry風という一般印象だけで仕様を決めない。
- 実装前に必要な仕様調査を飛ばさない。
