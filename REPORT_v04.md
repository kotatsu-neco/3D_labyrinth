# 灰の目録 3Dダンジョン試作 v04 作業報告

## 今回の修正対象

ユーザー指摘:

- 扉のところが少しズレている感じがする。
- 扉を開くと扉が消えるので修正が必要。

UI全体は後回しという方針のため、今回は3Dダンジョン内の扉表示だけを修正対象にしました。

## 変更内容

### 1. 扉配置の対称化

`app.js` の `addDoor()` を作り直しました。

- セル中心 `centerX` / `centerZ` を基準に配置。
- 左右枠、上枠、扉板を対称に計算。
- 以前の `margin * 0.55` / `CELL - margin * 0.91` のような非対称指定を廃止。
- 南北方向通路の扉、東西方向通路の扉の両方に対応。

### 2. 開放後の扉表示

以前は、扉が開くと `tile === TILE.DOOR && !isDoorOpen(...)` の条件により、扉ジオメトリ全体を描画しなくなっていました。

v04では以下に変更しました。

- 扉タイルでは常に `addDoor(...)` を呼ぶ。
- 閉じている場合は、中央に閉扉板を描画。
- 開いている場合は、扉枠を残し、左右に開いた扉板を描画。

該当箇所:

- `buildSceneGeometry()`
- `addDoor(g, gridX, gridZ, doorColor, frameColor, frameDark, isOpen)`

### 3. 簡易マップ表示

- 閉じた扉: `D`
- 開いた扉: `O`

開いた扉が完全な通常床に見えないよう、デバッグ表示上も区別しました。

### 4. キャッシュ対策

`index.html` のCSS/JS参照を以下へ更新しました。

- `style.css?v=20260609-v04`
- `app.js?v=20260609-v04`

画面タイトル/HUD/初期メッセージも `v04` に更新しました。
