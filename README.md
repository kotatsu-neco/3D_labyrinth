# 暗澹の石櫃 v34b

v34bでは、v34aの街画面UIの器を、確定した `town_spec_v01e` の街機能方針に合わせて再整理した。
街機能の本実装ではなく、Codex実装前に、宿屋・寺院・商店・酒場・訓練場の画面入口と操作導線がWizardry型の仕様から外れないようにするためのUI設計修正版である。

GitHub Pages等で配布する場合は、キャッシュ更新のため `style.css` と `app.js` に `20260612-v34b` のクエリを付けている。

## v34b 変更点

- `BUILD_VERSION` を `v34b` に更新。
- `index.html` の `<title>` と画面上部HUDを `暗澹の石櫃 v34b` に更新。
- キャッシュバスターを `20260612-v34b` に更新。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v34b` に更新。
- キャンプ呪文画面のCSS基準クラスを `spell-camp-v34b-frame` に更新。
- 街トップ注記を、確定済みの施設役割に合わせて更新。
- 宿屋を「個人単位の宿泊」UIに変更。
- 宿屋に部屋ランク選択の器を追加。
- 寺院を「処置対象選択」→「支払者選択」の導線が見えるUIに変更。
- 寺院に `お金を集める` の器を追加。
- 商店を `買う` / `売る` / `鑑定` / `解呪` / `お金を集める` / `街へ戻る` のUIに変更。
- 商店の各操作は、まず対象キャラクターを選ぶUIへ進む。
- 商店の `お金を集める` は、買い物担当者へ金を集中させる補助機能として画面に置いた。
- 酒場注記を、金分配=均等配分として明確化。
- 訓練場注記を、個人管理・後続でパーティー解散確認が必要であることへ更新。
- `docs/DUNGEON_ACTION_SPEC_v34b.md` と `docs/reports/REPORT_v34b.md` を追加。

## v34bで実装しないこと

- 酒場の実際の加入/離脱/均等分配。
- 訓練場の実際の新規作成/転職/引退削除。
- 宿屋の実際の支払い、HP回復、呪文回復、レベルアップ判定。
- 寺院の実際の支払者選択処理、金集中、蘇生/処置判定、1d52週加齢。
- 商店の実際の購入/売却/鑑定/解呪/金集中。
- 複数パーティーの本格状態モデル移行。

## 操作上の前提

- 起動時は街画面。
- `迷宮へ` は休止中パーティーがない場合のみ現行固定パーティーで出発する。
- 休止中パーティーがある場合、v34bでは `冒険を再開する` へ誘導する。
- 全滅パーティーは自動帰還しない。
- 救助・回収は、後続実装でキャンプの `周辺を探索する` に接続する。
