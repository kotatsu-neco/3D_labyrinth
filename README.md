# 暗澹の石櫃 v29a

スマホ向け3DダンジョンRPG試作です。  
英語表記: **Stone Casket of Gloom**

v29aでは、v29の装備変更実装を維持し、装備入口から所持品画面へ進んだ場合の戻り先保持を修正しました。

v29では、v28までの戦闘中対象選択とキャンプ内アイテム操作を維持し、次段階として **装備変更** を接続しました。キャンプ内のアイテム選択から `装備 / 外す` を実行でき、装備状態が所持品一覧・装備一覧・ステータス詳細・戦闘判定へ反映されます。

GitHub Pages等で配布する場合は、キャッシュ更新のため `style.css` と `app.js` に `20260612-v29a` のクエリを付けています。

## v29a 変更点

- `BUILD_VERSION` を `v29a` に更新。
- `index.html` の `<title>` と画面上部HUDを `暗澹の石櫃 v29a` に更新。
- キャッシュバスターを `20260612-v29a` に更新。
- 戦闘ウィンドウのCSS基準クラスを `battle-window-v29a` に更新。
- 所持品を実行時にアイテム個体へ正規化し、各個体に装備状態を持たせるようにした。
- キャンプ → アイテム → メンバー → アイテム選択に `装備 / 外す` を追加。
- キャンプの `装備` 画面からメンバーを選び、所持品画面へ進めるようにした。
- 装備入口から所持品画面へ入った場合、戻り先が通常アイテム選択ではなく装備一覧になるようにした。
- 装備中アイテムを `[E]` または `E` 表示で分かるようにした。
- 装備可能アイテムは `WEAPON / ARMOR / SHIELD / HEAD` と表示し、装備不可アイテムは装備ボタンを無効化した。
- 武器装備を `memberDamageDice(member)` と命中補正へ反映した。
- 防具・盾・兜装備を `effectiveMemberAc(member)` へ反映し、探索カード・キャンプ内一覧・戦闘パーティ表示・ステータス詳細・敵攻撃命中判定で参照するようにした。
- 装備中アイテムを渡す・捨てる場合は、処理内で装備状態を解除してから移動または削除するようにした。

## v29aで変更していないこと

- アイテムはスタックしません。
- `quantity` / `weight` は使っていません。
- 外部JSONの実行時読み込みはまだ行っていません。
- 敵名・アイテム名・呪文名はまだ仮名であり、正本化していません。
- 職業別装備制限、両手武器、二刀流、アクセサリ、呪い、鑑定、売買はまだ行っていません。
- 戦闘外での呪文使用はまだ行っていません。
- `DOWN` 状態の蘇生はまだ行いません。

## 起動方法

`index.html` をブラウザで開いてください。

GitHub Pagesなどにアップロードする場合は、zipの中身をそのまま配置します。

## 実機確認で見てほしい点

- キャンプ → 装備 → メンバー選択 → 所持品選択、の流れが分かりやすいか。
- キャンプ → アイテムからも `装備 / 外す` に自然に進めるか。
- 装備中アイテムの `[E]` / `E` 表示が分かりやすいか。
- 装備不可アイテムのボタン無効表示が分かりやすいか。
- 装備変更後、探索カード・ステータス詳細・戦闘画面のAC表示が自然に更新されるか。
- iPhone Safariでアイテム操作画面のボタン密度が窮屈ではないか。
